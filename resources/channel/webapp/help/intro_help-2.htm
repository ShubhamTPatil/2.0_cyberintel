Policy Manager enables you, as an administrator, to assign packages to targets. These targets represent one or more endpoints, such as users, machines, or groups. For each target, you can assign a state, such as "stage," "install," or "uninstall," to the package. The state you specify is applied to the package at each target following a schedule that you set. Policy Manager is browser-based and stores information about packages and targets in a directory service, using the LDAP protocol.

This document is intended for network managers, system administrators, help-desk personnel, and IT managers who deploy and configure Policy Manager or use it to manage a network of computers. This document assumes that you are familiar with BMC Configuration Management (CM) products, such as tuners and transmitters.

This product consists of the following components:

    • Policy Manager channel, which provides the browser-based interface you use to create policies for distributing packages to targets, assign states to packages, and schedule when packages are downloaded and installed. You also use this component to specify configuration settings. Policy Manager also provides a command-line interface for configuring some of the settings used for distributing packages to targets.
    • Common Management Services channel, which allows you to configure system-wide settings that apply to Policy Manager and other BMC CM web applications you might have.
    • Policy Service channel, which is the client-side component and is responsible for applying the policies assigned to the target tuner on which it is running.

Additionally, you need one of the following directory services:

    • Microsoft Active Directory (AD)
    • Microsoft Active Directory Application Mode (ADAM)
    • Sun ONE Directory

For information about which versions of these directory services are supported, see the Policy Management release notes, available on the BMC Customer Support web site.
