You can view the policy compliance information for targets and packages using any of the following tabs:

    • The Target View tab
    • The Target Compliance tab
    • The Package Compliance tab

If your target contains members (such as groups, containers, and organizational units), you can view two types of compliance information on the Target Compliance tab:

    • The Policy Compliance tab shows compliance information for policies directly and indirectly assigned to the target. Queries you run here consider the latest LDAP-synced policies. For more information about LDAP synchronization, see Setting up the console server for policy compliance.
    • The Overall Compliance tab shows compliance information for all the machines in the target, regardless of the policies from which a given machine receives its packages.

      • Overall compliance queries are useful for tracking deployments. They run faster than the more robust compliance reports, in part because they do not consider recently-published LDAP-synced policies. When you want to focus on inventory scan performance and find out how many machines are checking in, use overall compliance queries. Use the compliance report when you require synchronization with the latest policy information. For more information about the compliance report, see Compliance reports.

Locating targets and running queries

To locate a target and run a compliance query

  1. Locate a target on the Target View tab.

    2. Alternatively, you can begin on the Target Compliance tab.

    • In the left pane, browse and select the target for which you want to view compliance information.

  2. Click Compliance to display compliance information.

    3. The target name appears on the Target Compliance tab at the top of the right pane. If your target contains members, you can view policy compliance or overall compliance. The Policy Compliance tab is active by default.

Viewing policy compliance for targets

You can view compliance information for targets that include members, or targets that are individual machines. The procedures that follow apply to any target.

To view policy compliance for a target

  1. To view compliance information for all assigned packages in your selected target:
    • Locate a target on the Target Compliance tab:
    • In the left pane, browse and select the target for which you want to view compliance information. The target name appears at the top of the right pane, and the Policy Compliance tab is active.
    • Select all packages by selecting the check box in the header row, and click Calculate.

  2. To view compliance information for specific packages you have assigned to the selected target:
    1. On the Policy Compliance tab, select the check box for each individual package you want to include in your query.
    2. Click Calculate.
  3. To view information about endpoints that are compliant, non-compliant, or not checked-in:
    1. Click the pound sign (#) in the Compliance column header to see the exact number of endpoints for each status.
    2. Click the percent sign (%) in the Compliance column header to see the number of endpoints as a percentage of the total number of endpoints in the target.

      3. The numbers or percentages appear in colors that indicate the status:

    3. Green for compliant.
    4. Red for non-compliant.
    5. Blue for not checked-in.

      6. Click the displayed numbers or percentages themselves to view a list of machines related to packages with a given status.

  4. To view compliance information for all machines in the selected target:
    1. Click the Overall Compliance tab. The green, red, and blue compliance status tabs represent the machines in the selected target that are compliant, non-compliant, and not checked-in, respectively.
    2. Click the green, red, or blue compliance status tabs to view the list of machines related to each status.

    3. The Green tab displays all the machines in this target that are compliant with all policies that you have assigned to them, both directly and indirectly.
    4. The Red tab displays all the machines in this target that are not compliant with all policies you have assigned to them, both directly and indirectly. The list displays one cause of non-compliance for each machine.

      5. Note: The list displays only one cause of non-compliance for performance reasons. Click the machine name to view more information about a non-compliant machine.

    5. The Blue tab displays all the machines in this target that have not checked in since you made any policy changes that affect the machines. For each machine, the list displays the date when the machine last reported status.

      6. Click the link for any machine name to review details about the machine or (with appropriate privileges) to initiate tuner administration.

      Reviewing query details

      To review the setup and results of the query in Report Center, click the Show Report Center Query button.

      Generating a report

      To generate an overall compliance report for the target, click the Generate Compliance Report button. Confirm the message window, and the Compliance Reports tab appears. For more information, see Compliance reports.

Viewing policy compliance for packages

You can view policy compliance information for a package to find out:

    • What targets should have this package?
    • Do those targets have this package?

Viewing package compliance is similar to viewing target compliance, except you use the Package Compliance tab to query targets related to a package, instead of the Target Compliance tab to query packages related to a target.

The Search region enables you to find and filter the list of packages you want to include in your query by searching on the package name or date of publication. The search function supports the asterisk (*) and percent sign (%) as wildcards. See To view policy compliance for a target.
