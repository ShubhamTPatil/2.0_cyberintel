As an initiative of Green IT, endpoint users shutdown their PCs during non-working hours. Because of this, all the deployments scheduled while the PCs are turned off resume when users turn on their PCs at the beginning of a working day. This causes high usage in network bandwidth due to the downloading of channels as specified in the policy.

In order to avoid this network congestion, you can use the Policy Management Wake-on-Wan (WoW) feature to wake up machines during non-peak network periods so that deployments can be made reliably and without network disruption.

WoW deployments look for a tuner on the subnet to use as a "seed" tuner. A seed tuner is the first available tuner on the subnet which is used to distribute a "magic packet" to wake up WoW targets.

WoW prerequisites

    • The motherboards of WoW target machines must support the Wake-On-LAN (WoL) feature.
    • You must perform an LDAP Sync to identify member machines.
    • WoW seed tuners must be version 7.5 or 7.2.02 with the Power Management Service channel (7.2.02 only).
    • WoW target machines and seed tuner machines must be known to the inventory database.

      • Note: ˝˝˝˝˝˝˝˝˝˝˝˝˝˝˝˝˝˝˝˝ﬂﬂﬂﬂ The WoW feature cannot wake machines that shutdown abnormally. For example, if the machine is unplugged before being shut down, you may not be able to wake the machine using the WoW feature.

To create a WoW deployment

  1. Choose a target in Policy Manager and add channels.
  2. Click to select the WoW Deployment option.
  3. Set the primary and secondary schedules for the policy.
  4. Save the policy.

    5. If the endpoint is powered off, the WoW deployment wakes up the machine based on the set schedule.
