Eine Blackout-Periode definiert einen Zeitraum im Verlauf des Tages, in dem Sie nicht möchten, dass Policy Management den Status von Packages an einem oder mehreren Zielen verändert. Die Blackout-Periode verhindert alle Downloads, Installationen, Aktualisierungen und Reparaturen innerhalb des angegebenen Zeitraums. Sie können auch angeben, dass keine Blackout-Periode eingerichtet werden soll, wenn Sie möchten, dass Aktivitäten zu jeder Zeit zugelassen sind. Anweisungen zur Einrichtung von Blackout-Perioden finden Sie unter Festlegen der Blackout-Periode für ein Ziel.

Hinweis: Während der angegebenen Blackout-Periode werden sämtliche Aktivitäten für alle Packages auf dem Client-Tuner unterbunden. Dies gilt auch für Packages, die nicht über eine Subscription verwaltet werden. Berücksichtigen Sie dies beim Einrichten der Blackout-Periode, da es möglicherweise zu Notfallsituationen kommen kann, in denen Sie ein Package während einer Blackout-Periode auf Clients aktualisieren müssen. In diesem Fall müssen Sie entweder warten, bis die Blackout-Periode vorbei ist, oder Aktualisierungen an jedem Client manuell vornehmen.
