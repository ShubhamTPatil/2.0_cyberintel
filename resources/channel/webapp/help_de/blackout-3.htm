Legen Sie anhand der folgenden Schritte eine Blackout-Periode fest, die standardmäßig verwendet wird, um Änderungen am Policy-Status an einem oder mehreren Zielen zu verhindern. Die Blackout-Periode verhindert alle Downloads, Installationen, Aktualisierungen und Reparaturen innerhalb des angegebenen Tageszeitraums. Normalerweise verhindern Sie mit der Blackout-Periode mögliche Störungen der Werktagsaktivitäten an den Clients oder minimieren den Netzwerkverkehr zu kritischen Tageszeiten. Aktivitäten, deren Ausführung während der Blackout-Periode geplant ist, erfolgen sofort nach Beendigung dieser. Die Blackout-Periode ist relativ zur Zeitzone des Clients.

Hinweis: Während der angegebenen Blackout-Periode werden sämtliche Aktivitäten für alle Packages auf dem Client-Tuner unterbunden. Dies gilt auch für Packages, die nicht über Policy Management verwaltet werden. Berücksichtigen Sie dies beim Einrichten der Blackout-Periode, da es möglicherweise zu Notfallsituationen kommen kann, in denen Sie ein Package während einer Blackout-Periode auf Clients aktualisieren müssen. In diesem Fall müssen Sie entweder warten, bis die Blackout-Periode vorbei ist, oder Aktualisierungen an jedem Client manuell vornehmen.

Globale Blackout-Periode. Um eine globale Blackout-Periode einzurichten, wählen Sie das Ziel "Alle Clients", bevor Sie die Blackout-Periode angeben. Sind zwei Blackout-Perioden für einen Client eingerichtet (wenn zwei Policys auf denselben Client angewendet werden), dann setzt sich die Blackout-Periode des Clients aus diesen beiden Blackout-Perioden zusammen. Benutzer Alan beispielsweise gehört zur Benutzergruppe Engineering, und sowohl für Alan als auch für die Benutzergruppe ist eine Blackout-Periode eingerichtet. Die Blackout-Periode für Alan gilt von 7:00 Uhr bis 11:00 Uhr, die Blackout-Periode für "Engineering" von 9:00 Uhr bis 17:00 Uhr. Daraus ergibt sich für Alan eine Blackout-Periode von 7:00 Uhr bis 17:00 Uhr.

Policy Service vom Blackout ausnehmen. Beim Einrichten einer Blackout-Periode können Sie den Policy Service ausnehmen, so dass dieser während einer Blackout-Periode weiterhin aktiv ist und Aktualisierungen durchführen kann. Die Ausnahme ist dann nützlich, wenn Sie bei der Aktualisierung oder Installation von Packages während einer Blackout-Periode Notfallsituationen erwarten.

Packages vom Blackout ausnehmen. Auf der Registerkarte "Packages" der Seite "Policy bearbeiten" können Sie angeben, welche Packages von der Blackout-Periode ausgenommen werden sollen. Die Ausnahme ist dann nützlich, wenn Sie bei der Aktualisierung oder Installation von Packages während einer Blackout-Periode Notfallsituationen erwarten. Weitere Informationen dazu finden Sie unter Ausnehmen von Packages von der Blackout-Periode.

Wenn mehrere Benutzer (bei Verwendung des gleichen Policy Manager von zwei unterschiedlichen Computern aus) die Seite "Blackout-Periode" für das gleiche Ziel bearbeiten, dann werden die letzten zu speichernden Änderungen übernommen.

Hinweis: Einstellungen auf Zielebene, wie zum Beispiel die Blackout-Periode, können jeweils nur für ein Ziel festgelegt werden. Sie können keine Einstellungen auf Zielebene einrichten, wenn Sie sich im Mehrfachauswahlmodus befinden und gleichzeitig Policys für mehrere Ziele bearbeiten. Sie können jedoch für alle Ziele Einstellungen auf Zielebene einrichten, indem Sie das Ziel Alle Clients auswählen.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So legen Sie die Blackout-Periode fest:

  1. Klicken Sie auf der Seite "Policy bearbeiten" auf die Registerkarte Blackout-Periode.

    2. Die Seite "Blackout-Periode" wird angezeigt.

  2. Wählen Sie im Bereich Blackout-Periode eine der folgenden Optionen:
    • Wenn Sie eine Blackout-Periode festlegen möchten, wählen Sie Folgende Blackout-Periode festlegen, und geben Sie im Bereich Aktivitäten deaktivieren die Start- und Endzeiten für die Blackout-Periode an.

      • Hinweis: Blackout-Perioden müssen auf einen Zeitraum innerhalb eines Tages festgelegt werden (zum Beispiel von 9:00 Uhr bis 17:00 Uhr) und dürfen nicht über Mitternacht hinausgehen (zum Beispiel 18:00 Uhr bis 5:00 Uhr).

    • Wenn Sie keine Blackout-Periode festlegen möchten, dann wählen Sie Keine. Aktivitäten zu jedem Zeitpunkt zulassen.
  3. Wenn Sie für das Ziel eine Blackout-Periode festgelegt haben, dann wählen Sie im Bereich Policy Service-Verhalten eine der folgenden Optionen:
    • Aktualisierung von Policy Service während der Blackout-Periode und Vorgänge für von der Blackout-Periode ausgenommene Packages zulassen.
    • Aktualisierung von Policy Service während der Blackout-Periode nicht zulassen. Keine Package-Vorgänge durchführen, auch nicht für die von der Blackout-Periode ausgenommenen Packages.
  4. Klicken Sie nach dem Erstellen oder Bearbeiten der Policy auf Vorschau.

    5. Auf der Seite "Vorschau" können Sie die Änderungen an der Policy noch einmal überprüfen, bevor Sie diese speichern und auf das Ziel anwenden. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.

  5. Nachdem Sie sich die Änderungen der Policy noch einmal angesehen haben, klicken Sie auf Speichern, um die Änderungen zu bestätigen und zu speichern.

Informationen über die Blackout-Periode

Wenn Sie eine Blackout-Periode angeben, dann hängt Policy Service den von Ihnen angegebenen Zeitraum an die Tunereigenschaft Marimba.Zeitplan.Filter in der Datei prefs.txt des Client-Tuners an. Besagt zum Beispiel der ursprüngliche Aktualisierungszeitplan, dass der Client-Tuner jederzeit und an einem beliebigen Wochentag aktualisiert werden kann, und Sie legen mit Hilfe von Policy Management eine Blackout-Periode von 9:00 Uhr bis 17:00 Uhr fest, dann lautet der Wert der Tuner-Eigenschaft wie folgt:

marimba.schedule.filter=ANYTIME on mon+tue+wed+thu+fri+sat+sun BLACKOUT 9:00AM-5:00PM

Wenn Sie später die Option "Keine Blackout-Periode" wählen, dann wird der ursprünglich auf dem Client-Tuner festgelegte Channel-Aktualisierungszeitplan verwendet. In dem genannten Beispiel wird nur BLACKOUT 9:00AM-5:00PM aus dem Wert von marimba.schedule.filter entfernt. Die Tuner-Eigenschaft, die die Blackout-Periode betrifft (marimba.schedule.filter), wird auf dem Client nicht gelöscht. Sie können diese Tuner-Eigenschaft explizit über die Seite "Tuner- und Package-Eigenschaften" löschen. Weitere Informationen dazu finden Sie unter Einstellen der Tuner- und Package-Eigenschaften für ein Ziel.
