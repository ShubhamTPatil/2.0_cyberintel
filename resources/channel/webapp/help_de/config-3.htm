Üblicherweise müssen Sie das Plugin jedes Mal konfigurieren und veröffentlichen, wenn Sie Policy Manager installieren oder aktualisieren. Allerdings müssen Sie das Plugin auch während der folgenden Situationen neu veröffentlichen:

    • Wenn Sie die Einstellungen auf der Seite "Plugin-Konfiguration" ändern.
    • Wenn Sie die Einstellungen mit dem Befehl -setpluginparam ändern.
    • Wenn Sie die Attribute des Policy Management-Objekts mit dem Befehl -configSet ändern.

      • Hinweis: Sie müssen das Policy Service-Plugin nicht jedes Mal veröffentlichen, wenn Sie Policys verändern.

Wenn möglich, entnimmt Policy Manager den Systemeinstellungen (Applikationen > Konsole > Systemeinstellungen > Datenquelle > Verzeichnisdienste) die folgenden Verzeichnisdienstinformationen und gibt diese automatisch auf der Seite "Plugin-Konfiguration" ein.

    • Hostname und Portnummer des Verzeichnisdienstes
    • Base DN
    • Bind DN
    • SSL verwenden oder nicht

Sind diese Informationen bereits auf der Seite eingetragen, dann müssen Sie bestätigen, dass es sich um die Einstellungen handelt, die Sie verwenden möchten.

Hinweis: Wenn Sie bereits vorher schon einmal das Policy Service-Plugin auf einem Transmitter veröffentlicht haben, und SSL war aktiviert, starten Sie Policy Manager neu, bevor Sie das Plugin auf einem Transmitter ohne SSL veröffentlichen.

So konfigurieren und veröffentlichen Sie das Policy Service-Plugin:

  1. Klicken Sie auf die Registerkarte Konfiguration.
  2. Klicken Sie auf der Konfigurationsseite auf den Link Plugin.

    3. Die Seite "Plugin-Konfiguration" wird angezeigt. Links oben auf der Seite sehen Sie Datum und Uhrzeit, zu denen das Plugin das letzte Mal veröffentlicht wurde.

  3. Geben Sie im Bereich Master Transmitter Informationen zur Veröffentlichung des Plugins an:
    1. Geben Sie die URL-Adresse für den Policy Service-Channel an, in dem Sie das Plugin veröffentlichen möchten.

      2. Hinweis: Die URL-Adresse muss auf SubscriptionService enden.

    2. Ist eine Veröffentlichungsberechtigung erforderlich, geben Sie Veröffentlichungsbenutzername und Passwort an, die für die Veröffentlichung der Konfigurationsänderungen auf dem Transmitter benötigt werden, der das Plugin hostet.

      3. Hinweis: Der Veröffentlichungsbenutzername und das entsprechende Passwort sind nur dann erforderlich, wenn Sie (oder der Primäradministrator Ihres Unternehmens) die Transmitter so konfiguriert haben, dass für die Veröffentlichung von Channels und Packages Benutzernamen und Passwörter abgefragt werden.

  4. Geben Sie im Bereich Verzeichnisdienste die benötigten Informationen ein.

    5. Normalerweise sind die Felder in diesem Bereich bereits mit Informationen aus dem für CMS angegebenen Verzeichnisdienst ausgefüllt. Die einzigen Felder, die Sie eventuell ändern müssen, sind die Felder für den Bind DN und das Bind DN-Passwort, wenn Sie zum Beispiel einen Bind DN verwenden möchten, dessen Berechtigungen sich von denen des für CMS angegebenen DN unterscheiden.

    Informationen zu den Feldern auf dieser Seite finden Sie unter Plugin-Konfigurationsseite: Verzeichnisdienstfelder.

  5. Abhängig vom verwendeten Verzeichnisdienst können Sie eventuell die Verbindung zwischen Policy Manager und dem Verzeichnisdienst durch Klicken auf die Schaltfläche Start neben Testserver verifizieren.

    6. Policy Manager stellt eine Verbindung zu dem von Ihnen angegebenen Verzeichnisdienst her und überprüft, ob der Server und der Port Verbindungen mit dem festgelegten Benutzernamen und Passwort akzeptieren. Policy Manager überprüft dann, ob das Subscription-Konfigurationsobjekt vorhanden ist. Das Subscription-Konfigurationsobjekt befindet sich im Container für Konfigurationsobjekte (ist dem BMC CM-Container untergeordnet). Dieses Objekt wird erstellt, wenn Sie das Installationsskript ausführen oder LDIF-Skripts aktualisieren.

    Hinweis: Der Computer, auf dem Policy Manager installiert ist, muss nicht auf den Verzeichnisdienst zugreifen können. Daher kann das Policy Service-Plugin möglicherweise trotz Fehlschlagen dieses Tests erfolgreich eine Verbindung herstellen.

  6. Wenn Sie Policy Manager mit neu bereitgestellten Computern verwenden möchten, wählen Sie das Kontrollkästchen Verwendung von Policys als Modell für neu bereitgestellte Computer zulassen. Weitere Informationen finden Sie im Kapitel über die Integration der Tuner-Installationen in die Bereitstellung von Betriebssystemen im BMC CM-Deployment-Handbuch, das auf der BMC Customer Support-Website verfügbar ist.
  7. Klicken Sie auf Vorschau.

    8. Die Seite "Vorschau der Plugin-Änderungen" wird angezeigt. Diese Seite hebt die von Ihnen auf der Seite "Plugin-Konfiguration" gemachten Änderungen hervor, so dass Sie diese noch einmal vor der Veröffentlichung überprüfen können. Standardmäßig werden alle Einstellungen mit hervorgehobenen Änderungen angezeigt. Um nur die Änderungen anzuzeigen, klicken Sie auf den Link Nur Änderungen anzeigen

  8. Nachdem Sie Ihre Änderungen überprüft und gegebenenfalls modifiziert haben, klicken Sie auf Veröffentlichen.

    9. Wenn Sie auf dieser Seite auf "Veröffentlichen" klicken, dann veröffentlichen Sie das Policy Service-Plugin für den Policy Service-Channel auf dem Transmitter und kehren zur Konfigurationsstartseite zurück.

Tipp: Wenn Sie überprüfen möchten, ob das Policy Service-Plugin veröffentlicht wurde, lassen Sie sich über Transmitter Administrator die Channel auf Ihrem Transmitter anzeigen, und erweitern Sie den Policy Service-Channel wie in Abbildung 2-1 dargestellt, um dessen Inhalt anzuzeigen. Wenn Sie das Segment .configurator sehen, wurde das Plugin auf dem Transmitter veröffentlicht.
