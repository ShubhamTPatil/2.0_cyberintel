Normalerweise sind die Verzeichnisdienstfelder auf der Seite "Plugin-Konfiguration" bereits mit Informationen aus dem für CMS angegebenen Verzeichnisdienst ausgefüllt. Die Liste der auszufüllenden Felder hängt von dem Verzeichnisdienst ab, den Sie verwenden. Die Seite "Plugin-Konfiguration" enthält die folgenden Verzeichnisdienstfelder:

    • Hostname und Port—Hostname und Port des Verzeichnisdienstes in der Form <Hostname>:<Port>. Der von Ihnen hier festgelegte Verzeichnisdienst wird zum Speichern von Policys verwendet. Standardmäßig entsprechen die Einträge dem für CMS angegebenen Verzeichnisdienst, der für die Anmeldeauthentifizierung verwendet wird. (Normalerweise muss dieses Feld für Active Directory nicht ausgefüllt werden.)

      • Stellt Ihr Unternehmen replizierte Verzeichnisdienste zum Speichern von Policys zur Verfügung, verwenden Sie eine Verzeichnisdienst-zu-Repeater-Zuordnung. Sie können den Vorteil replizierter Verzeichnisdienste ausnutzen, indem Sie Policy Management so konfigurieren, dass jeder Repeater einen nahegelegenen Verzeichnisdienst kontaktiert und nicht denjenigen, der dem Master Transmitter zugewiesen wurde. Zudem können Sie jedem Repeater eine Liste mit Verzeichnisdiensten zuweisen. Tritt bei einem der Verzeichnisdienste aus der Liste ein Fehler auf, dann versucht der Repeater den nächsten zu kontaktieren. Dadurch wird verhindert, dass bei Ausfall eines Verzeichnisdienstes das gesamte System beeinträchtigt wird. Informationen über das Zuordnen von Transmittern zu Verzeichnisdiensten finden Sie im Administratorhandbuch für Policy Management. beschriebene Befehlszeilenoption -ldapservers verwenden.

    • SSL-Verbindung zwischen Transmitter und Verzeichnisdienst verwenden: Aktivieren Sie dieses Kontrollkästchen, um Secure Sockets Layer (SSL) für eine sichere Kommunikation zwischen dem Transmitter und dem Verzeichnisdienst zu aktivieren. Weitere Informationen dazu finden Sie im Abschnitt über Sicherheit und SSL im BMC CM-Administratorhandbuch für die Infrastruktur (Infrastructure Administrator's Guide), das auf der BMC Customer Support-Website verfügbar ist.
    • Base DN: Grundlegender Distinguished Name für die Verzeichnisdienstverbindung. Der Base DN definiert den Umfang der Verzeichnisansicht für das Plugin. In den meisten Fällen entspricht dieser dem Sun ONE Directory-Suffix. (Normalerweise muss dieses Feld für Active Directory nicht ausgefüllt werden).

      • Beispiel: Verwenden Sie für ADAM oder Sun ONE Directory dc=company,dc=com.

    • Bind DN: Der Distinguished Name des Benutzerkontos, welches das Policy Service-Plugin beim Erstellen einer Verbindung zum Verzeichnisdienst verwendet. Dieses Benutzerkonto sollte Leseberechtigungen für den Bereich des durch den Base DN-Eintrag festgelegten Verzeichnisses haben. Dieser Bereich wird normalerweise dem gesamten Verzeichnisbaum zugeordnet.

      • Verwenden Sie für Active Directory eines der folgenden Formate:

    • Der vollständige Distinguished Name (DN), wie zum Beispiel cn=Administrator,cn=Users,dc=company,dc=com
    • Der Benutzerprinzipalname (UPN), wie zum Beispiel Administrator@company.com

      • Verwenden Sie für ADAM den vollständigen Distinguished Name (DN), wie zum Beispiel cn=Administrator,cn=Users,dc=company,dc=com

      Verwenden Sie für Sun ONE Directory eines der folgenden Formate:

    • Der vollständige Distinguished Name (DN), wie zum Beispiel uid=Administrator,ou=People,dc=company,dc=com
    • Der allgemeine Name (CN) für den Verzeichnisadministrator, wie zum Beispiel cn=Directory Manager
    • Bind DN-Passwort—Das für den Zugriff auf den Verzeichnisdienst benötigte Passwort. Dieses Passwort wird vom Plugin für die Verzeichnisdienstverbindung verwendet.
    • Größe des Verbindungspools für Verzeichnisdienst: Das Policy Service-Plugin verwendet einen Pooling-Mechanismus zum Erstellen und Aufrechterhalten von Verbindungen mit dem Verzeichnisdienst. Dieser Parameter legt die maximale Anzahl der im Pool hergestellten Verbindungen fest und kann normalerweise auf dem Standardwert von 25 belassen werden.
    • Die Gültigkeitsdauer (in Minuten) der letzten erfolgreichen Hostverbindung: Wenn Sie für Verzeichnisdienst eine Liste mit Failover-Hostnamen angeben, geht Policy Manager die Liste solange durch, bis erfolgreich eine Verbindung zu einem Host hergestellt wurde. Policy Manager verwendet diese erfolgreiche Hostverbindung, bis die von Ihnen festgelegte Gültigkeitsdauer abgelaufen ist. Danach wird erneut versucht, eine Verbindung zum ersten Hostnamen aus der Liste herzustellen.
    • Verwendung von Policys als Modelle für neu bereitgestellte Computer zulassen: Wenn Sie die Tuner-Installation in die Bereitstellung der Betriebssysteme integrieren, können Sie durch Markieren dieses Kontrollkästchens Gruppen im Verzeichnisdienst identifizieren, deren Policy Sie als Modell für die Installation neuer Applikationen und Inhalte auf den neu bereitgestellten Computern verwenden möchten. Weitere Informationen finden Sie im Kapitel über die Integration der Tuner-Installationen in die Bereitstellung von Betriebssystemen im BMC CM-Deployment-Handbuch, das auf der BMC Customer Support-Website verfügbar ist.
