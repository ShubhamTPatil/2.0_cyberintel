In diesem Kapitel wird beschrieben, wie Sie Policy Manager mit der Patch Management-Lösung von BMC CM verwenden können, um Patches auf den Zielcomputern bereitzustellen.

Folgende Themen werden behandelt:

  • Voraussetzungen für die Integration mit Patch Management
  • Was ist eine Patch-Gruppe?
  • Was ist ein Patch-Gruppe-Zuweisungsstatus?
  • Was ist Patch Service?
  • Zuweisen von Patch-Gruppen zu Zielen
  • Simulation der Patch-Installation
  • Anzeigen weiterer Informationen zur Installation von Patch-Gruppen
  • Konfigurieren der Neustarteinstellungen für Patch-Installationen
  • Außerkraftsetzen des Patch Service-Aktualisierungszeitplans für Zielcomputer.
  • Ausnehmen von Patch Service von der Blackout-Periode.
  • Policy Compliance für Patch-Gruppen

Mit Hilfe der Patch Management-Lösung von BMC CM kann festgestellt werden, welche Patches von den Computern in Ihrem Unternehmen benötigt werden. Weiterhin können damit Patches gruppiert, konfiguriert und auf einem Transmitter veröffentlicht werden. Verwenden Sie anschließend Policy Manager, um Patch-Gruppen für die Computer bereitzustellen, die sie benötigen. Weitere Informationen über die Patch Management-Lösung von BMC CM finden Sie im Administratorhandbuch für Patch Management, das auf der BMC CM-Website des Customer Supports erhältlich ist.
