Eine Patch-Gruppe ist eine Liste aus einer oder mehreren Patch-IDs, die jeweils einen Patch eindeutig bezeichnen und weitere Informationen zu dem Patch enthalten. Sie können Patch-Gruppen für einzelne Clients oder für benutzerdefinierte Gruppen von Clients erstellen. Patch-Gruppen bieten ein großes Ausmaß an Flexibilität beim Testen und Verteilen von Patches. Sie können beispielsweise eine Gruppe von Sicherheits-Patches erstellen, die nur auf Clients der Vertriebsmitarbeiter bereitgestellt werden soll.

Das Zuweisen von Patch-Gruppen zu Zielcomputern in Policy Manager erfolgt ähnlich wie das Zuweisen von Packages zu Zielcomputern. Wählen Sie einen oder mehrere Zielcomputer aus, bearbeiten Sie die Policys, und fügen Sie die Patch-Gruppen hinzu, die Sie zuweisen möchten.

Patch-Gruppen werden durch das Patch-Gruppe-Symbol gekennzeichnet. Wenn angezeigt werden soll, welche Patches Teil einer Patch-Gruppe sind, klicken Sie auf den Namen der Patch-Gruppe (wird normalerweise als blauer Link angezeigt).

Hinweis: Die Mitglieder einer Patch-Gruppe werden nur angezeigt, wenn die Patch-Gruppe auf dem bei der Konfiguration von Patch Manager festgelegten Master Transmitter veröffentlicht wurde.
