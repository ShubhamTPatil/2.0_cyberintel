Die Installation einiger Patches macht möglicherweise einen Neustart der Zielcomputer erforderlich. Verwenden Sie in Policy Manager die Optionen auf der Registerkarte „Neustarteinstellungen“, um festzulegen, in welchem Umfang Sie den Benutzern Steuerungsmöglichkeiten gewähren möchten, wenn die Computer nach dem Erhalt eines Patches neu gestartet werden müssen. Mit diesen Optionen werden die Eigenschaften festgelegt, die das Neustartverhalten steuern. Weitere Informationen zu den Eigenschaften finden Sie unter Erweitert: Tuner-Eigenschaften für die Neustarteinstellungen.

Bei dieser Vorgehensweise wird davon ausgegangen, dass Sie sich auf der Seite „Policy bearbeiten“ befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So verwenden Sie Policy Manager, um die Neustarteinstellungen für Patch-Installationen zu verwalten

  1. Klicken Sie auf die Registerkarte Patches >Neustarteinstellungen.

    2. Die Registerkarte „Neustarteinstellungen“ wird angezeigt.

  2. Wählen Sie Policy Manager für die Verwaltung der Patch-Neustarteinstellungen verwenden.
  3. Wählen Sie eine oder mehrere der folgenden Optionen aus:
    • Automatischen Neustart nach Patch-Installation zulassen – Wählen Sie diese Option aus, wenn Sie möchten, dass der Zielcomputer automatisch nach der Installation von Patches, die einen Neustart erfordern, neu gestartet wird.

      • Wenn den Benutzern Warnmeldungen bezüglich des Neustarts angezeigt werden sollen, konfigurieren Sie die folgenden Optionen:

    • Vor dem Neustart Warnmeldungen für Benutzer anzeigen – Wählen Sie diese Option aus, wenn der Zielcomputer Meldungen anzeigen soll, die den Benutzer über den Neustart informieren.
    • Aufschieben des Neustarts durch Endbenutzer nicht zulassen... - Wählen Sie diese Option aus, wenn die Benutzer über den Neustart informiert werden, jedoch keine Möglichkeit haben sollen, den Neustart aufzuschieben. Der in der folgenden Option beschriebene Neustart-Countdown beginnt sofort nach der Installation der Patches.
    • Aufschieben des Neustarts durch Endbenutzer zulassen für insgesamt _ Minuten - Wählen Sie diese Option, wenn Sie das Aufschieben des Neustarts durch Benutzer zulassen möchten. Geben Sie die Anzahl an Minuten an, für die Benutzer einen Neustart aufschieben dürfen. Nach Ablauf dieser Zeit können Benutzer einen Neustart nicht mehr aufschieben, und der Countdown für den Neustart wird gestartet.
    • Countdown für Neustart – Legen Sie die Wartezeit bis zum Neustart des Computers fest. Der Countdown für den Neustart gibt den Benutzern Zeit, ihre Arbeiten vor dem Neustart zu speichern.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.
