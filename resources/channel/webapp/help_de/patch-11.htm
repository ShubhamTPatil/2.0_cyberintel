Dieser Abschnitt dient der Referenz und soll Sie dabei unterstützen, die unter Konfigurieren der Neustarteinstellungen für Patch-Installationen beschriebenen Verfahren durchzuführen. Das Konfigurieren der Neustarteinstellungen beeinflusst folgende Tuner-Eigenschaften:

    • marimba.reboot.interact - Legt fest, ob den Benutzern eine Warnmeldung bezüglich des Neustarts angezeigt werden soll. Der Standardwert ist "true" (eine Warnmeldung wird angezeigt).
    • marimba.reboot.interact.allowSnooze - Legt fest, ob Benutzer einen Neustart aufschieben dürfen. Wenn auf „true“ gesetzt, wird ein Dialogfeld angezeigt, das es dem Benutzer ermöglicht, einen Neustart aufzuschieben. Wenn die Eigenschaft auf "false" festgelegt ist, wird ein Dialogfeld angezeigt, in dem die verbleibende Zeit als Countdown bis zum Neustart angezeigt wird. Der Standardwert ist "true".
    • marimba.reboot.interact.snooze.maxTime - Legt die maximale Dauer (in Millisekunden) fest, innerhalb derer es dem Benutzer möglich sein soll, einen Neustart aufzuschieben. Der Standardwert ist 0 (null), keine maximale Zeitspanne.
    • marimba.reboot.interact.timer - Legt die Wartezeit oder die Countdown-Zeit vor einem Neustart in Sekunden fest. Wenn marimba.reboot.interact.allowSnooze auf "false" festgelegt wird, gilt dieser Wert als Startzeit und wird im Warndialogfeld angezeigt. Der Standardwert ist 60 Sekunden. Wenn Sie diesen Wert auf 0 setzen, wird der Computer sofort neu gestartet.
    • marimba.reboot.never - Legt fest, ob der Computer bei einem erforderlichen Neustart automatisch neu gestartet wird oder nicht (z. B. nach einer Patch-Installation). Legen Sie den Wert auf "true" fest, wenn der Computer nicht automatisch neu gestartet werden soll.

Diese Eigenschaften werden Ihnen möglicherweise auf der Registerkarte „Erweitert“ > „Tuner/Package-Eigenschaften“ angezeigt, wenn Sie eine Policy bearbeiten.
