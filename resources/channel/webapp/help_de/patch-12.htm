Unter diesem Thema wird beschrieben, wie Sie mit Policy Manager den Patch Service-Aktualisierungszeitplan außer Kraft setzen können. Standardmäßig befolgt Patch Service den mit Patch Manager festgelegten Aktualisierungszeitplan. Weitere Informationen zum Festlegen des Standard-Aktualisierungszeitplans für Patch Service finden Sie in der Online-Hilfe zu Patch Manager.

Während der Aktualisierung sucht Patch Service auf dem Ziel nach der Liste mit den erforderlichen und bereits installierten Patches. Außerdem werden den Abhängigkeiten jedes Patches entsprechend eine Reihe weiterer Patches installiert. Sie können angeben, dass Aktualisierungen anhand eines wiederkehrenden Zeitplans durchgeführt werden. Wenn Sie mit Policy Manager keinen Zeitplan festlegen, wird der mit Patch Manager festgelegte Zeitplan verwendet. Wird Patch Service einem Ziel nicht direkt als Teil der Policy zugewiesen, geht Policy Manager davon aus, dass Patch Service von dem gleichen Transmitter wie Patch Manager stammt.

Wenn Sie für das Ziel eine Blackout-Periode festgelegt haben, können Sie Patch Service davon ausnehmen. Die Ausnahme ist dann nützlich, wenn Sie Notfallsituationen erwarten, in denen Patches auch während einer Blackout-Periode installiert oder aktualisiert werden sollen. Weitere Informationen dazu finden Sie unter Ausnehmen von Patch Service von der Blackout-Periode..

Hinweis: Einstellungen auf Zielebene, wie zum Beispiel der Aktualisierungszeitplan für Patch Service, können jeweils nur für ein Ziel festgelegt werden. Sie können keine Einstellungen auf Zielebene einrichten, wenn Sie sich im Mehrfachauswahlmodus befinden und gleichzeitig Policys für mehrere Ziele bearbeiten. Sie können jedoch für alle Ziele Einstellungen auf Zielebene einrichten, indem Sie das Ziel Alle Clients auswählen.

Bei dieser Vorgehensweise wird davon ausgegangen, dass Sie sich auf der Seite „Policy bearbeiten“ befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So setzen Sie den Standard-Akualisierungszeitplan für Patch Service außer Kraft

  1. Klicken Sie auf der Seite „Policy bearbeiten“ auf die Registerkarte Patches > Patch Service-Zeitplan.

    2. Die Seite „Patch Service-Zeitplan“ wird angezeigt.

  2. Wählen Sie Policy Manager zum Festlegen des Patch Service-Aktualisierungszeitplans verwenden.
  3. Geben Sie im Bereich Wiederholung an, wie häufig Patch Service aktualisiert werden soll:
    • Täglich – Patch Service kann jeden Tag, an Wochentagen (Montag bis Freitag) oder alle 1 bis 365 Tage aktualisiert werden, je nachdem, welchen Wert Sie festlegen.
    • Wöchentlich – Patch Service kann alle 1 bis 52 Wochen aktualisiert werden, je nachdem, welchen Wert Sie festlegen. Dabei wird die Aktualisierung in der festgelegten Woche nur an dem ausgewählten Tag durchgeführt. Sie müssen mindestens einen Wochentag auswählen.
    • Monatlich – Patch Service kann alle 1 bis 12 Monate an dem festgelegten Tag aktualisiert werden. Wenn Sie den 30. oder 31. wählen und der Monat der Aktualisierung keinen solchen Tag hat (wie Februar), wird die Aktualisierung am ersten Tag des Folgemonats durchgeführt.
  4. Geben Sie im Abschnitt Aktualisierungszeit die Tageszeit an, zu der Patch Service aktualisiert werden soll:
    • Aktual. um: Legen Sie die Uhrzeit fest, zu der die Aktualisierung erfolgen soll.
    • Aktual. alle: Legen Sie fest, wie häufig am Tag eine Aktualisierung erfolgen soll. Sie können festlegen, dass Aktualisierungen in einem bestimmten Intervall von Minuten oder Stunden am Tag der Aktualisierung stattfinden sollen. Sie können die Aktualisierung aber auch für einen bestimmten Zeitraum innerhalb des Tages festlegen. Aktualisierungen können beispielsweise jede Stunde zwischen 9:00 und 17:00 Uhr erfolgen oder alle 30 Minuten zwischen 17:00 und 9:00 Uhr.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.
