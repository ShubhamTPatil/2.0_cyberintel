Wenn Sie eine Blackout-Periode festgelegt haben, innerhalb derer Downloads und Installationen verhindert werden sollen, sollten Sie Patch Service mit Hilfe von Policy Manager davon ausnehmen.

Ausnehmen von Patch Service von der Blackout-Periode. Beim Festlegen einer Blackout-Periode können Sie den Patch Service davon ausnehmen, so dass dieser während einer Blackout-Periode weiterhin ausgeführt wird und aktualisieren kann. Die Ausnahme ist dann nützlich, wenn Sie Notfallsituationen erwarten, in denen während einer Blackout-Periode Patches installiert werden sollen.

Ausnehmen von Patch-Gruppen von der Blackout-Periode. Auf der Seite „Policy bearbeiten“ auf der Registerkarte „Patch-Gruppen“ können Sie angeben, welche Patch-Gruppen von der Blackout-Periode ausgenommen werden sollen. Diese Ausnahme ist nützlich, wenn Sie Notfallsituationen erwarten, in denen Patches so schnell wie möglich bereitgestellt werden sollen, auch wenn sich einige Clients zu diesem Zeitpunkt in einer Blackout-Periode befinden. Weitere Informationen dazu finden Sie unter Zuweisen von Patch-Gruppen zu Zielen.

Hinweis: Das Ausnehmen von Patch-Gruppen von der Blackout-Periode wird nur bei der ersten Bereitstellung von Patch-Gruppen auf den Zielcomputern angewendet. Patch Service verwaltet Installationen und andere Aktionen, die nach der Bereitstellung von Patch-Gruppen stattfinden.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite „Policy bearbeiten“ auf der Registerkarte „Packages“ befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So nehmen Sie Patch Service von der Blackout-Periode aus

  1. Klicken Sie auf der Seite „Policy bearbeiten“ auf die Registerkarte Blackout-Periode.

    2. Die Seite „Blackout-Periode“ wird angezeigt.

    Hinweis: Die Optionen in Schritt 2 werden nur dann angezeigt, wenn eine Blackout-Periode festgelegt wurde.

  2. Wählen Sie eine der folgenden Optionen aus, um das Patch Serice-Verhalten festzulegen:
    • Aktualisierung von Patch Service während der Blackout-Periode und Vorgänge für von der Blackout-Periode ausgenommene Patch-Gruppen zulassen.
    • Aktualisierung von Patch Service während der Blackout-Periode nicht zulassen. Keine Vorgänge für Patch-Groups durchführen, auch nicht für die von der Blackout-Periode ausgenommenen.
  3. Klicken Sie nach dem Erstellen oder Bearbeiten der Policy auf Vorschau.

    4. Auf der Seite „Vorschau“ können Sie die Änderungen an der Policy noch einmal überprüfen, bevor Sie sie speichern und auf das Ziel anwenden. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.

  4. Nachdem Sie sich die Änderungen der Policy noch einmal angesehen haben, klicken Sie auf Speichern, um die Änderungen zu bestätigen und zu speichern.
