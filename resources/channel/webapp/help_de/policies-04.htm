Eine Policy erstellen Sie, indem Sie ein Ziel auswählen und anschließend die Packages angeben, die dem Ziel zugewiesen werden sollen. Sie können für jedes Package folgende Informationen angeben:

    • Primär- und Sekundärstatus
    • Primär-, Sekundär-, Aktualisierungs- und Reparaturzeitplan
    • Installationspriorität
    • Blackout-Ausnahme

Zusätzlich können Sie in einer Policy folgende Informationen angeben:

    • Status und Zeitpläne für die Packages
    • Blackout-Periode eines Ziels und die Ausnahme des Policy Service von der Blackout-Periode
    • Aktualisierungszeitplan für Policy Service auf dem Ziel
    • Tuner- und Package-Eigenschaften, die für das Ziele festlegt werden sollen
    • Für das Ziel erforderliche Transmitter-Berechtigungen
    • Profilinformationen für das Ziel

      • Hinweis: Wenn Sie sich im Mehrfachauswahlmodus befinden und gleichzeitig Policys für mehrere Ziele bearbeiten, können Sie nur Packages und Package-Informationen bearbeiten, wie beispielsweise Status, Zeitpläne, Installationspriorität und Blackout-Ausnahmen. Package-Status, Zeitpläne, Installationspriorität und Blackout-Ausnahmen, die Sie speichern, gelten für alle von Ihnen angegebenen Ziele. Die Registerkarten für die Bearbeitung anderer Informationen, wie beispielsweise Blackout-Zeitplan und Aktualisierungszeitplan für Policy Service, werden nur dann angezeigt, wenn Sie die Policy für ein einzelnes Ziel bearbeiten.

Das folgende Verfahren beschreibt, wie eine Policy von der Seite "Zielansicht" aus erstellt oder bearbeitet wird. Sie können eine Policy auch von den folgenden Seiten aus erstellen oder bearbeiten:

    • Zielinformationen
    • Package-Ansicht
    • Package-Informationen

Wenn Sie die Erstellung oder Bearbeitung einer Policy über diese Seiten vornehmen, unterscheidet sich nur der erste Schritt im folgenden Verfahren. Die verbleibenden Schritte sind identisch.

So erstellen oder bearbeiten Sie eine Policy:

  1. Wählen Sie auf der Policy Manager-Seite "Zielansicht" das Ziel aus, für das eine Policy erstellt oder bearbeitet werden soll, und klicken Sie danach auf Bearbeiten.

    2. Die Seite "Policy bearbeiten" wird angezeigt. Auf der Registerkarte "Packages" werden die dem Ziel zugewiesenen Packages angezeigt.

  2. So fügen Sie Packages zur Policy hinzu, oder entfernen diese:
    1. Klicken Sie auf Package-Liste bearbeiten.
    2. Sie fügen ein Package hinzu, indem Sie links auf der Seite einen Transmitter angeben, der als Host für die Packages fungiert, oder indem Sie ihn aus der Liste der aktuell bereitgestellten Packages auswählen. Wählen Sie anschließend ein Package aus, so dass es rechts auf der Seite angezeigt wird. Weitere Informationen dazu finden Sie unter Hinzufügen von Packages zu einer Policy.
    3. Um ein Package zu entfernen, wählen Sie dieses im rechten Bereich der Seite aus. Klicken Sie anschließend auf Entfernen. Weitere Informationen dazu finden Sie unter Entfernen von Packages aus einer Policy.
    4. Klicken Sie nach dem Hinzufügen und Entfernen der Packages auf OK.
  3. Geben Sie auf der Seite "Policy bearbeiten" auf der Registerkarte Packages die Status und Zeitpläne für die Packages an, die Sie der Policy hinzugefügt haben. Weitere Informationen dazu finden Sie unter Festlegen von Status und Zeitplänen für Packages in einer Policy.
  4. Geben Sie auf der Seite "Policy bearbeiten" auf der Registerkarte Packages die Installationspriorität für die Packages in der Policy an. Weitere Informationen dazu finden Sie unter Festlegen der Installationspriorität für Packages in einer Policy.
  5. Geben Sie auf der Seite "Policy bearbeiten" auf der Registerkarte Packages die Packages an, die von der Blackout-Periode ausgenommen werden sollen. Weitere Informationen dazu finden Sie unter Ausnehmen von Packages von der Blackout-Periode.
  6. Wenn Sie eine Policy für ein einzelnes Ziel erstellen oder bearbeiten, können Sie folgende Tasks ausführen:
    • Festlegen einer Blackout-Periode für ein Ziel
    • Angeben des Policy Service-Zeitplans für ein Ziel
    • Einstellen der Tuner- und Package-Eigenschaften für ein Ziel
    • Festlegen von Transmitter-Berechtigungen für ein Ziel
    • Festlegen des Profils für ein Ziel
  7. Klicken Sie nach dem Erstellen oder Bearbeiten der Policy auf Vorschau.

    8. Auf der Seite "Vorschau" können Sie sich die Änderungen der Policy noch einmal ansehen, bevor Sie diese speichern und auf das Ziel anwenden. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.

  8. Nachdem Sie sich die Änderungen der Policy noch einmal angesehen haben, klicken Sie auf Speichern, um die Änderungen zu bestätigen und zu speichern.

Wenn Policy Service das nächste Mal aktualisiert und auf dem Client ausgeführt wird, wird die erstellte oder bearbeitete Policy heruntergeladen und angewendet.
