Der Installationsstatus bestimmt, ob und wie ein Package auf einem Ziel installiert wird. Sie können für jedes einem Ziel zugewiesene Package einen Primäsrstatus festlegen. Mit Hilfe von Policy Manager können Sie einem Package als Teil der Policy für einen oder mehr Clients Installationsstatus zuweisen.

In der folgenden Tabelle werden die für Packages verfügbaren, unterschiedlichen Status beschrieben. Mit den beiden Primärstatus Bereitstellen und Ankündigen können Sie einen Sekundärstatus festlegen, mit dem die Installation des Package gesteuert werden kann. Sie können mit dem Sekundärstatus einen Aktivierungszeitplan verknüpfen, um für die Zukunft einen Zeitraum zu bestimmen, innerhalb dessen Package-Installationen im gesamten Unternehmen durchgeführt werden. Dies gilt selbst für Clients, die nur selten eine Verbindung zum Netzwerk herstellen. Der Standardwert des Sekundärstatus ist Installieren.

Konfliktlösung: Status und Zeitpläne in Policys

Statusvorrang

Zwischen den verschiedenen Package-Status können Konflikte auftreten, wenn ein Client in zwei oder mehr Gruppen vertreten ist oder wenn eine Policy sowohl einem einzelnen Client als auch den Mitgliedern einer Gruppe zugewiesen ist, zu der der Client gehört. In diesen Fällen gelten die folgenden Vorrangsregeln. Policy Manager setzt den Status mit dem höchsten Vorrang durch.

  1. Ausschließen (höchster Rang)
  2. Deinstallieren
  3. Primär
  4. Installieren-Starten-Beibehalten
  5. Installieren-Starten
  6. Installieren-Beibehalten
  7. Installieren
  8. Bereitstellen
  9. Ankündigen (niedrigster Rang)

    10. Hinweis: Konflikte treten zwischen verschiedenen Package-Status auf und resultieren nicht aus der Art, wie eine Policy einem Ziel zugewiesen wird. Daher beruht die Konfliktlösung nicht darauf, ob ein Client ein direktes Ziel oder als Mitglied einer Gruppe ein indirektes Ziel ist. Weitere Informationen dazu finden Sie unter Konfliktlösung: Status und Zeitpläne in Policys.
