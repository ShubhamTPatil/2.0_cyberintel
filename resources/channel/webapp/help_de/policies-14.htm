Sie können den Zeitplan für Primär- und Sekundärstatus von Packages festlegen. Wenn eine Installation geplant ist, lädt der Ziel-Tuner das Package vom Transmitter herunter und führt die von Ihnen angegebenen Installationsaktivitäten aus. Wenn Sie sowohl Primär- und Sekundärstatus festgelegt haben, führt der Ziel-Tuner die Installationsaktivitäten zur geplanten Zeit aus. Zeitpläne beziehen sich auf die Zeitzone des Clients.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So legen Sie den Zeitplan für Primär- und Sekundärstatus von Packages fest:

  1. Suchen Sie auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" die Zeilen der Packages, für die Zeitpläne festgelegt werden sollen.
  2. Klicken Sie in der Zeile des Package auf den Pfeil in der ersten Spalte links auf der Seite, um die Zeitplaninformationen für das Package anzuzeigen.
  3. So legen Sie einen Zeitplan für den Primärstatus fest:
    1. Klicken Sie im Abschnitt Zeitplan für Primärstatus auf Bearbeiten.

      2. Die Seite "Primärzeitplan bearbeiten" wird angezeigt.

    2. Geben Sie in den Bereichen Aktivierung und Verfall Zeiten an, um einen Zeitraum zu bestimmen, innerhalb dessen Packages für den Primärstatus verfügbar sein sollen.

      3. Weitere Informationen dazu finden Sie unter Was sind Aktivierung und Verfall?.

    3. Klicken Sie auf OK, um den Zeitplan zu speichern und zur Seite "Policy bearbeiten" zurückzukehren.

      4. Hinweis: Der Zeitplan wird jetzt im Abschnitt "Zeitplan für Primärstatus" angezeigt.

  4. Wenn Sie einen Sekundärstatus für das Package angegeben haben, legen Sie folgendermaßen einen Zeitplan für den Sekundärstatus fest:
    1. Klicken Sie im Abschnitt Zeitplan für Sekundärstatus auf Bearbeiten.

      2. Die Seite "Sekundärzeitplan bearbeiten" wird angezeigt.

    2. Geben Sie in den Bereichen Aktivierung und Verfall Zeiten an, um einen Zeitraum zu bestimmen, innerhalb dessen Packages für den Sekundärstatus verfügbar sein sollen.

      3. Weitere Informationen dazu finden Sie unter Was sind Aktivierung und Verfall?.

    3. Klicken Sie auf OK, um den Zeitplan zu speichern und zur Seite "Policy bearbeiten" zurückzukehren.

      4. Beachten Sie, dass der Zeitplan jetzt im Abschnitt "Zeitplan für Sekundärstatus" angezeigt wird.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.

Hinweis: Sie können keinen Primär- und Sekundärzeitplan festlegen, bei dem der Sekundärzeitplan zeitlich vor dem Primärzeitplan liegt. Folgende Zeitpläne sind beispielsweise nicht gültig:

Primärzeitplan: Aktivierung am 02.01.04, 01:30 Uhr

Sekundärzeitplan: Aktivierung am 01.01.04, 24:00 Uhr
