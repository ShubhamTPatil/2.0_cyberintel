Sie können den Aktualisierungszeitplan für Packages festlegen. Wenn eine Aktualisierung geplant ist, überprüft der Ziel-Tuner, ob das Package auf dem Transmitter aktualisiert wurde, von dem das Package Aktualisierungen erhält. Wenn das Package bereits aktualisiert wurde, werden alle neuen oder veränderten Dateien heruntergeladen und auf dem Ziel installiert. Der Aktualisierungszeitplan hat lediglich dann Auswirkungen auf den Client, wenn bereits eine Subscription für das Package eingerichtet oder wenn das Package bereits installiert ist.

Hinweis: Für den Aktualisierungszeitplan des Package wird Nie angezeigt, wenn Sie ihn mit Tuner Administrator anzeigen. Dies gilt selbst dann, wenn mit Policy Manager ein anderer Zeitplan festgelegt wurde. Policy Service legt für den Aktualisierungszeitplan des Package Nie fest, so dass das Package sich nicht selbst aktualisiert, sondern nach den mit Policy Manager festgelegten Zeitplänen aktualisiert wird.

Es kann sein, dass Packages bereits ein Aktualisierungszeitplan zugewiesen wurde, so dass Sie entweder diesen Aktualisierungszeitplan benutzen oder ihn mit Hilfe von Policy Manager überschreiben können:

    • Die Erstellung des Package für die Software sowie die Softwareverteilung kann von unterschiedlichen Benutzern vorgenommen werden. Wenn Sie verantwortlich sind, den Policy Manager zur Verteilung der Software zu verwenden, können Sie den Zeitplan außer Kraft setzen, der von denjenigen, die Packages für die Software erstellen, festgelegt wurde.
    • Sie können sich auch entscheiden, während der Erstellung des Package einen Standardzeitplan festzulegen und diesen anschließend, wenn Sie Software an bestimmte Gruppen verteilen, außer Kraft setzen.
    • Sobald Sie den Zeitplan während der Erstellung des Package festgelegt haben, können Sie ihn nicht einfach ändern. Es ist einfacher, Zeitpläne mit Hilfe von Policy Manager zu ändern.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So legen Sie den Aktualisierungszeitplan für ein Package fest

  1. Suchen Sie auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" die Zeilen der Packages, für die Zeitpläne festgelegt werden sollen.
  2. Klicken Sie in der Zeile des Package auf den Pfeil in der ersten Spalte links auf der Seite, um die Zeitplaninformationen für das Package anzuzeigen.
  3. Klicken Sie im Abschnitt Aktualisierungszeitplan auf Bearbeiten.

    4. Die Seite "Aktualisierungszeitplan bearbeiten" wird angezeigt.

  4. Verwenden Sie die Dropdown-Liste, um den Zeitplan auszuwählen, dem das Package folgen soll:
    • Den bei Erstellung und Veröffentlichung des Package angegebenen Zeitplan befolgen. Wählen Sie diese Option aus, wenn Sie einen Zeitplan für das Package angegeben haben, als dieses erstellt und veröffentlicht wurde, und wenn Sie diesem Zeitplan folgen möchten.
    • Policy Manager zum Festlegen des Aktualisierungszeitplans verwenden. Wählen Sie diese Option aus, wenn Sie den Aktualisierungszeitplan mit Policy Manager verwalten möchten.
  5. Wenn Sie Policy Manager verwenden möchten, um den Aktualisierungszeitplan zu verwalten, geben Sie folgende Informationen an:
    1. Geben Sie im Bereich Wiederholung an, wie häufig das Package aktualisiert werden soll:
      • Nie: Das Package wird nie automatisch aktualisiert. Sie müssen das Package, wenn erforderlich, manuell aktualisieren.
      • Täglich: Das Package wird an jedem Wochentag (Montag bis Freitag) oder alle 1 bis 365 Tage aktualisiert, je nachdem, welchen Wert Sie festlegen.
      • Wöchentlich: Das Package kann alle 1 bis 52 Wochen aktualisiert werden, je nachdem, welchen Wert Sie festlegen. Dabei wird die Aktualisierung in der festgelegten Woche nur an dem ausgewählten Tag durchgeführt. Sie müssen mindestens einen Wochentag auswählen.
      • Monatlich: Das Package kann alle 1 bis 12 Monate an dem festgelegten Tag aktualisiert werden. Wenn Sie den 30. oder 31. wählen und der Monat der Aktualisierung keinen solchen Tag hat (wie Februar), wird die Aktualisierung am ersten Tag des Folgemonats durchgeführt.
    2. Geben Sie im Bereich Aktualisierungszeit die Uhrzeit an, zu der das Package aktualisiert werden soll:
      • Aktual. um: Legen Sie die Uhrzeit fest, zu der die Aktualisierung erfolgen soll.
      • Aktual. alle: Legen Sie mehrere Uhrzeiten fest, zu denen Aktualisierungen erfolgen sollen. Sie können festlegen, dass Aktualisierungen in einem bestimmten Intervall von Minuten oder Stunden am Tag der Aktualisierung stattfinden sollen. Sie können die Aktualisierung aber auch für einen bestimmten Zeitraum innerhalb des Tages festlegen. Aktualisierungen können beispielsweise zwischen 9:00 und 17:00 Uhr jede Stunde erfolgen oder zwischen 17:00 und 9:00 Uhr alle 30 Minuten.
    3. Geben Sie in den Bereichen Aktivierung und Verfall die Zeitpunkte an, ab und bis wann diesem Aktualisierungszeitplan gefolgt werden soll.

      4. Weitere Informationen dazu finden Sie unter Was sind Aktivierung und Verfall?.

  6. Klicken Sie auf OK, um den Zeitplan zu speichern und zur Seite "Policy bearbeiten" zurückzukehren.

    7. Hinweis: Der Zeitplan wird jetzt im Abschnitt "Aktualisierungszeitplan" angezeigt.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.
