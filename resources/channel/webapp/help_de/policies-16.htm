Applikationen und Inhalte, die Sie packen und an Benutzer verteilen, können manchmal unbenutzbar oder nicht korrekt ausgeführt werden. Sie können die Reparaturfunktionen dazu verwenden, automatisch nach Problemen, wie fehlenden oder fehlerhaften Dateien, zu suchen und diese zu beheben.

Sie können den Reparaturzeitplan für Packages festlegen. Bei einer geplanten Reparatur werden die Dateien neu installiert, die sich seit der Installation des Package geändert haben. Bevor das Package repariert wird, stellt der Ziel-Tuner eine Verbindung mit dem Transmitter her, von dem das Package Aktualisierungen erhält.

Der Reparaturzeitplan hat lediglich dann Auswirkungen auf den Client, wenn bereits eine Subscription für das Package eingerichtet oder wenn das Package bereits installiert ist. Außerdem funktionieren Reparaturvorgänge nur bei Packages, die mit BMC CM Application Packager erstellt wurden. Es kann sein, dass Packages bereits ein Reparaturzeitplan zugewiesen wurde, so dass Sie diesen Reparaturzeitplan verwenden oder ihn mit Policy Manager außer Kraft setzen können. Im Folgenden sind einige Situationen genannt, in denen Sie Policy Manager zum Festlegen eines Reparaturzeitplans verwenden sollten:

    • Die Erstellung des Package für die Software sowie die Softwareverteilung kann von unterschiedlichen Benutzern vorgenommen werden. Wenn Sie verantwortlich sind, den Policy Manager zur Verteilung der Software zu verwenden, können Sie den Zeitplan außer Kraft setzen, der von denjenigen, die Packages für die Software erstellen, festgelegt wurde.
    • Sie können sich auch entscheiden, während der Erstellung des Package einen Standardzeitplan festzulegen und diesen anschließend, wenn Sie Software an bestimmte Gruppen verteilen, außer Kraft setzen.
    • Sobald Sie den Zeitplan während der Erstellung des Package festlegen, können Sie ihn nicht einfach ändern. Es ist einfacher, Zeitpläne mit Hilfe von Policy Manager zu ändern.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So legen Sie den Reparaturzeitplan für ein Package fest:

  1. Suchen Sie auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" die Zeilen der Packages, für die Zeitpläne festgelegt werden sollen.
  2. Klicken Sie in der Zeile des Package auf den Pfeil in der ersten Spalte links auf der Seite, um die Zeitplaninformationen für das Package anzuzeigen.
  3. Klicken Sie im Abschnitt Reparaturzeitplan auf Bearbeiten.

    4. Die Seite "Reparaturzeitplan bearbeiten" wird angezeigt.

  4. Verwenden Sie die Dropdown-Liste, um den Zeitplan auszuwählen, dem das Package folgen soll:
    • Den bei Erstellung und Veröffentlichung des Package angegebenen Zeitplan befolgen. Wählen Sie diese Option aus, wenn Sie einen Zeitplan für das Package angegeben haben, als dieses erstellt und veröffentlicht wurde, und wenn Sie diesem Zeitplan folgen möchten.
    • Policy Manager zum Festlegen des Reparaturzeitplans verwenden. Wählen Sie diese Option aus, wenn Sie den Reparaturzeitplan mit Policy Manager verwalten möchten.
  5. Wenn Sie Policy Manager verwenden möchten, um den Reparaturzeitplan zu verwalten, geben Sie folgende Informationen an:
    1. Geben Sie im Bereich Wiederholung an, wie häufig das Package repariert werden soll:
      • Nie: Das Package wird nie automatisch überprüft und repariert. Sie müssen das Package, wenn erforderlich, manuell reparieren.
      • Täglich: Das Package wird an jedem Wochentag (Montag bis Freitag) oder alle 1 bis 365 Tage überprüft und repariert, je nachdem, welchen Wert Sie festlegen.
      • Wöchentlich: Das Package kann alle 1 bis 52 Wochen überprüft und repariert werden, je nachdem, welchen Wert Sie festlegen Anzahl..Dabei wird die Reparatur in der festgelegten Woche nur an den von Ihnen ausgewählten Tagen durchgeführt. Sie müssen mindestens einen Wochentag auswählen.
      • Monatlich: Das Package kann alle 1 bis 12 Monate an dem festgelegten Tag überprüft und repariert werden. Wenn Sie den 30. oder 31. wählen und der Monat der Reparatur keinen solchen Tag hat (z. B. Februar), wird die Reparatur am ersten Tag des Folgemonats durchgeführt.
    2. Geben Sie im Bereich Reparaturzeit die Uhrzeit an, zu der das Package repariert werden soll.
      • Reparieren um: Legen Sie die Uhrzeit fest, zu der die Reparatur erfolgen soll.
      • Reparieren alle: Legen Sie mehrere Uhrzeiten fest, zu denen Reparaturen erfolgen sollen. Sie können festlegen, dass Reparaturen in einem bestimmten Intervall von Minuten oder Stunden am Tag der Reparatur stattfinden sollen. Sie können die Reparatur aber auch für einen bestimmten Zeitraum innerhalb des Tages festlegen. Reparaturen können beispielsweise zwischen 9:00 und 17:00 Uhr jede Stunde erfolgen oder zwischen 17:00 und 9:00 Uhr alle 30 Minuten.
    3. Geben Sie in den Bereichen Aktivierung und Verfall die Zeitpunkte an, ab und bis wann diesem Reparaturzeitplan gefolgt werden soll.

      4. Weitere Informationen dazu finden Sie unter Was sind Aktivierung und Verfall?.

  6. Klicken Sie auf OK, um den Zeitplan zu speichern und zur Seite "Policy bearbeiten" zurückzukehren.

    7. Beachten Sie, dass der Zeitplan jetzt im Abschnitt "Reparaturzeitplan" angezeigt wird.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.
