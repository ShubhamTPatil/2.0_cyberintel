In den Fällen, in denen sich die für Packages festgelegten Status und Zeitpläne von denen für die Ziele in einer Policy unterscheiden, müssen Sie angeben, ob Sie den Unterschied beibehalten oder eine Einstellung für alle Ziele verwenden möchten.

Wenn Unterschiede bestehen, die aufgelöst werden müssen, wird auf der Seite "Policy bearbeiten" folgende Meldung angezeigt:

Mit gekennzeichnete Packages und Zeitpläne haben unterschiedliche Status und Zeitpläne über die ausgewählten Ziele hinweg. Sie können diese Unterschiede beibehalten oder einen Status oder Zeitplan auswählen, der auf alle ausgewählten Ziele angewendet wird.

So lösen Sie Unterschiede in den Status und Zeitplänen eines Package auf:

  1. Suchen Sie auf der Seite "Policy bearbeiten" die Packages, die für die ausgewählten Ziele unterschiedliche Status oder Zeitpläne aufweisen.
  2. Wählen Sie auf der Registerkarte "Packages" in der Spalte "Primärstatus" oder "Sekundärstatus" eine der folgenden Optionen aus:
    • Unterschiede für Ziele beibehalten: Wählen Sie diese Option aus, wenn Sie für die Ziele unterschiedliche Package-Status beibehalten möchten, so etwa, wenn Sie für ein Ziel den Status Bereitstellen und für ein anderes den Status Installieren beibehalten möchten.
    • <Status>: Wählen Sie diese Option aus, wenn Sie einen einzigen Package-Status festlegen möchten, der von allen Zielen übernommen wird.
  3. Wählen Sie für Zeitpläne, die für die ausgewählten Ziele unterschiedlich sind, im Abschnitt „Zeitplan“ eine der folgenden Optionen aus:
    • Unterschiede im <Primärzeitplan, Sekundärzeitplan, Aktualisierungszeitplan, Reparaturzeitplan> für Ziele beibehalten: Wählen Sie diese Option aus, wenn Sie unterschiedliche Package-Zeitpläne für die Ziele beibehalten möchten, etwa, wenn ein Package auf jedem Ziel zu verschiedenen Zeiten für die Installation verfügbar sein soll.
    • Bearbeiteten <Primärzeitplan, Sekundärzeitplan, Aktualisierungszeitplan, Reparaturzeitplan> auf alle ausgewählten Ziele anwenden: Wählen Sie diese Option aus, wenn Sie einen Zeitplan für das Package festlegen möchten, der auf alle Ziele angewendet wird. Sie müssen den Zeitplan festlegen, der angewendet werden soll. Weitere Informationen dazu finden Sie unter Überblick über Package-Zeitpläne.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.
