Die Installationspriorität bestimmt, welches Package zuerst installiert wird, wenn für mehrere Packages auf einem Ziel die Installation zur gleichen Zeit geplant ist. Eine Anleitung zum Ändern der Installationspriorität finden Sie unter Überlegungen zur Installationspriorität.

Die Installationsprioritätsregeln sind Richtlinien, die jedoch keine bestimmte Installationsreihenfolge garantieren. Weitere Informationen dazu finden Sie unter Ausnahmen von der Installationspriorität.

Es können jedoch Voraussetzungs-Packages angegeben werden, die für die Installation von anderen Packages erforderlich sind. Dabei müssen jedoch die Best Practices in Bezug auf das Planen von Aktualisierungen berücksichtigt werden. Weitere Informationen dazu finden Sie unter Verwalten von Software, Daten und Aktualisierungen.

Installationspriorität – Konzepte

Wenn Sie ein Package einem Ziel zuweisen, lautet der standardmäßige numerische Wert, der der Installationspriorität des Packages zugewiesen wird, 99999. Sie können diesen Wert anzeigen, indem Sie mit Hilfe eines LDAP-Browsers/Editor-Tools das Package-Attribut mrbachannelorder im Policy-Objekt für das Ziel anzeigen, wie in Abbildung 4-5 dargestellt.

Wenn Sie die Installationspriorität (wie in Festlegen der Installationspriorität für Packages beschrieben) festlegen, ändert sich der numerische Wert des Attributs mrbachannelorder entsprechend der von Ihnen festgelegten Priorität. Der numerische Wert für das Attribut mrbachannelorder stimmt mit der Nummer überein, die auf der Seite "Zielinformationen" oder "Policy bearbeiten" in der Spalte "Installationspriorität" angezeigt wird.

Das Package mit dem niedrigsten Wert wird zuerst installiert. In diesem Beispiel wird das Package mit dem Wert 1 zuerst installiert, gefolgt von 2 und 3.

Ausnahmen von der Installationspriorität

In diesem Abschnitt werden Situationen beschrieben, in denen Policy Manager nicht der angegebenen Installationspriorität folgt.

Zeitüberschreitungen und Post-Installationsskripts. In der Regel wartet Policy Service, bis die Installation der Packages abgeschlossen ist (einschließlich der Post-Installationsskripts), bevor das nächste in der Installationspriorität angegebene Package installiert wird. Unter folgenden Bedingungen ignoriert Policy Manager die Installationspriorität:

    • Das Herunterladen und die Installation des Package dauern länger als die angegebene Zeitüberschreitung (siehe marimba.subscription.timeout). Der Standardwert für die Zeitüberschreitung beträgt 60 Minuten. Wenn es zur Zeitüberschreitung kommt, beginnt Policy Service mit der Installation des nächsten Package.
    • Sie haben das Package so konfiguriert, dass es das Post-Installationsskript als getrennten Prozess ausführt. In diesem Fall geht Policy Service davon aus, dass das Package bereits installiert ist, wenn das Post-Installationsskript ausgeführt wird, und startet die Installation des nächsten Package.

Abhängigkeiten. Sie können mit Application Packager Abhängigkeiten für Packages festlegen. Dazu gehören der verfügbare Festplattenspeicherplatz, der installierte RAM oder das Vorhandensein von anderen Packages vor der Installation. Wenn Sie für ein Package Abhängigkeiten festgelegt haben, haben diese Vorrang vor der Installationspriorität.

Überlegungen zur Installationspriorität

Wenn Sie die Installationspriorität für Packages festlegen, müssen Sie folgende Einstellungen der "Zielansicht" berücksichtigen, welche sich auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" befinden:

    • Startpriorität. Die Zahl, die Sie als Startpriorität angeben, bestimmt die Package-Installationsreihenfolge im Vergleich zu Packages in anderen Policys. Die Startpriorität ist dann wichtig, wenn bestimmte Clients zu mehreren Zielen oder Gruppen gehören, und Sie steuern möchten, ob die Packages, die einem Ziel zugewiesen werden, eine höhere Priorität besitzen als die Packages, die einem anderen Ziel zugewiesen werden. Beispiel: Wenn Sie möchten, dass Packages, die dem Ziel "Alle Clients" zugewiesen sind, eine höhere Priorität besitzen als diejenigen, die anderen Zielen oder Gruppen zugewiesen wurden, können Sie die Installationsprioritätswerte 1 bis 100 für das Ziel "Alle Clients" reservieren. Allen anderen Zielen können Sie dann eine Startpriorität von 101 und höher zuweisen.
    • Priorität. Die Reihenfolge, in der Packages auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" angezeigt werden, bestimmt deren Installationspriorität. Standardmäßig wird für die Priorität N/A angezeigt, was einem Attributwert für mrbachannelorder von 99999 entspricht.


    • Priorität aktualisieren      . Mit dieser Schaltfläche können Sie die Änderungen in der Priorität übernehmen, die Sie für Packages in das Textfeld Priorität eingegeben haben. Um die Priorität eines Packages in den Standardwert zu ändern (k. A. oder 99999), löschen Sie die Inhalte des Textfeldes "Priorität", und klicken Sie auf Priorität aktualisieren. Sie müssen immer auf die Schaltfläche Priorität aktualisieren klicken, bevor Sie auf "Vorschau" klicken. Nur so können Sie Ihre Änderungen speichern.
    • Priorität zurücksetzen. Mit dieser Schaltfläche können die letzten Prioritätsänderungen, die Sie vorgenommen haben, zurückgesetzt werden. Sobald Sie die Änderungen gespeichert haben, kann die Priorität nicht mehr zurückgesetzt werden.

Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

Festlegen der Installationspriorität für Packages

So legen Sie die Installationspriorität für Packages fest:

  1. Legen Sie auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" die Startpriorität für Packages in dieser Policy fest.
    1. Geben Sie in das Feld Startpriorität einen numerischen Wert ein, der die Priorität der Policy im Vergleich zu anderen Policys angibt. Weitere Informationen dazu finden Sie unter Installationspriorität – Konzepte.
    2. Klicken Sie auf Übernehmen.

      3. Beachten Sie, dass das erste Package in der Policy die Zahl erhält, die Sie im Textfeld Startpriorität angegeben haben, etwa 101. Die folgenden Packages erhalten die Zahlen entsprechend Ihrer Reihenfolge in der Policy (das zweite Package erhält 102, das dritte erhält 103 usw.).

  2. Wenn Sie die Installationspriorität eines Package erhöhen oder verringern möchten, suchen Sie die Zeile des Package.
  3. Geben Sie eine andere Zahl in das Feld "Priorität" ein, wenn Sie eine neue Installationspriorität zuweisen möchten.

    4. Hinweis: Beispiel: Wenn Sie eine 5 in das Feld "Priorität" eingeben und bereits über ein Package mit der Installationspriorität von 5 verfügen, erhöht Policy Manager solange die vorhandene und die folgenden Installationsprioritäten um 1, bis alle Zahlenkonflikte beseitigt wurden.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.
