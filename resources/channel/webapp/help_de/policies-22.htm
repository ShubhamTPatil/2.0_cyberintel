Beim Anzeigen der Seite "Zielinformationen" kann es vorkommen, dass verschiedene Packages die gleiche Installationspriorität besitzen. Sie legen beispielsweise folgende Installationsprioritäten für den Benutzer "Alan" fest:

    • Package A, Installationspriorität 1
    • Package B, Installationspriorität 2
    • Package C, Installationspriorität 3

Sie legen darüber hinaus folgende Installationsprioritäten für die Benutzergruppe "Engineering" fest:

    • Package D, Installationspriorität 1
    • Package E, Installationspriorität 2
    • Package F, Installationspriorität 3

Wenn "Alan" Mitglied der Gruppe "Engineering" ist, zeigt die Seite "Zielinformationen" sich überschneidende Installationsprioritäten an (zwei Mal 1, zwei Mal 2, zwei Mal 3). In diesen Fällen haben alle Packages mit Installationspriorität 1 Vorrang vor denjenigen mit Installationspriorität 2 usw. Sie können weitere Konflikte bei der Installationspriorität auch mit dem Wert der Startpriorität lösen, indem Sie Packages mit sich überschneidenden Zielen verschiedene Blöcke von Prioritätsnummern zuweisen.
