Wenn Sie eine Policy mit Hilfe von Policy Manager erstellen oder bearbeiten, können Sie auch die Tuner- und Package-Eigenschaften für ein Ziel festlegen. Mit Tuner- und Package-Eigenschaften können Sie das Verhalten des Ziels oder der Packages auf den Zielclients steuern. So können Sie beispielsweise die Anzahl und Häufigkeit von Wiederholungsversuchen bei einem fehlgeschlagenen Download oder das Neustartverhalten eines bestimmten Package steuern.

Dieser Abschnitt umfasst folgende Themen:

    • Was sind Tuner-Eigenschaften?
    • Welche Tuner-Eigenschaften werden üblicherweise mit Policy Manager festgelegt?
    • Was sind Package-Eigenschaften?
    • Eigenschaften zur Steuerung des Neustart von Packages (nur Windows)

Was sind Tuner-Eigenschaften?

Viele Eigenschaften und Einstellungen eines Tuner werden durch Tuner-Eigenschaften konfiguriert. Dies umfasst beispielsweise die URL-Adresse seines Primär-Channel, die URL-Adresse für Tuner-Aktualisierungen sowie Informationen zu Lizenzen und Proxies. Mit Hilfe von Policy Manager können Sie (in der Datei prefs.txt) Tuner-Eigenschaften für den Ziel-Tuner hinzufügen, bearbeiten und löschen.

Eine Liste von Tuner-Eigenschaften, die üblicherweise mit Policy Manager festgelegt werden, finden Sie unter Welche Tuner-Eigenschaften werden üblicherweise mit Policy Manager festgelegt?. Ein Liste von Tuner-Eigenschaften finden Sie im Kapitel über Tuner-Eigenschaften im BMC CM-Referenzhandbuch (BMC Configuration Management Reference Guide), das auf der BMC Customer Support-Website erhältlich ist.

Eine Anleitung zum Festlegen von Tuner-Eigenschaften finden Sie unter Festlegen von Tuner-Eigenschaften.

Welche Tuner-Eigenschaften werden üblicherweise mit Policy Manager festgelegt?

Wenn Sie Policy Service installiert haben, können Sie für den Tuner zusätzliche Eigenschaften einstellen. Mit einer der folgenden Vorgehensweisen können Sie die zusätzlichen Tuner-Eigenschaften festlegen, und so die Interaktion des Tuner mit Policy Service verändern.

    • Beim Erstellen eines Tuner-Installationsprogramms während Setup und Deployment
    • Über eine Profilaktualisierung
    • Über Tuner Administrator
    • Über Policy Manager

Die ersten neun Eigenschaften, die in der Tabelle 4-7 dargestellt sind, werden in der Liste auf der Policy Manager-Seite "Tuner- und Package-Eigenschaften" angezeigt. Sie sind in der Reihenfolge aufgelistet, in der sie auf der Seite angezeigt werden. Die übrigen Eigenschaften in der Tabelle müssen mit ihrem vollständigen Eigenschaftsnamen eingegeben werden. Sie sind in alphabetischer Reihenfolge aufgelistet.

marimba.subscription.nodelete
 
"true" oder "false"
 
Standard: false
Standardmäßig werden Packages gelöscht, wenn ein Benutzer oder ein Computer aus einer Gruppe im zu Grunde liegenden Verzeichnisdienst entfernt wird. Wenn Sie diese Eigenschaft auf "true" setzen, werden Packages nicht gelöscht. Dies gilt selbst dann, wenn Sie Packages mit Hilfe der Seite "Zielinformationen" vom Ziel löschen. Sie bleiben auf dem Ziel erhalten und werden nicht mehr mit Policy Manager verwaltet. Sie müssen den Status "Deinstallieren" verwenden, um die Packages vom Ziel zu löschen.
Neustart durchführen

"true" oder "false"
 
Standard: false
Bei "true" bestimmt Policy Service, ob nach der Anwendung der Policy ein Neustart erforderlich ist, und startet den Computer, falls erforderlich, neu.
 
Abbruch zulassen

"true" oder "false"
 
Standard: true
Standardmäßig wird in Policy Service im Dialogfeld „Neustart“ die Schaltfläche "Abbrechen" und daneben die Schaltfläche "Neustart jetzt durchführen" angezeigt. Wenn diese Eigenschaft auf "false" festgelegt ist, wird die Schaltfläche "Abbrechen" nicht angezeigt.
 
Neustartzeit
 
marimba.subscription.reboot.time
Integer
 
Standard: 300
(300 Sekunden oder 5 Minuten)
Diese Eigenschaft legt die Verzögerung in Sekunden fest, bevor Policy Service nach der Anwendung einer Policy das System neu startet.
Die Neustartzeit beginnt, sobald die Installation des letzten Package in der Policy abgeschlossen ist.
  • Wenn Sie eine negative ganze Zahl angeben (z. B. -1), gibt es keinen Countdown für einen Systemneustart. Client-Benutzern wird ein Dialogfeld angezeigt, in dem sie gefragt werden, ob sie mit dem Neustart fortfahren möchten oder nicht. (Benutzer können darauf mittels der Schaltflächen "OK" und "Abbrechen" antworten.)
  • Wenn Sie 0 (Null) angeben, sehen die Benutzer das Dialogfeld nur kurz aufleuchten (es wird angezeigt und verschwindet sofort wieder), und der Neustart beginnt unmittelbar.
  • Wenn Sie einen ungültigen oder keinen Wert angeben, wird der Standardwert von 300 Sekunden verwendet.

Integer

(60 Sekunden oder 1 Minute)
Diese Eigenschaft legt die Verzögerung in Sekunden fest, bevor Policy Service versucht, eine Subscription für fehlgeschlagene Packages einzurichten.

Diese Eigenschaft legt die Anzahl der Wiederholungen fest, bevor Policy Service nicht mehr versucht, eine Subscription für fehlgeschlagene Packages einzurichten.

marimba.subscription.useshortcuts
"true" oder "false"
 
Standard: false
Wenn diese Eigenschaft auf "true" gesetzt wird, werden angekündigte Packages als Desktopverknüpfungen (nur unter Windows) dargestellt.
Installationsmodus
 
marimba.subscription.installmode
"silent" oder "aspackaged"
 
Standard: Hintergrund (silent)
Diese Eigenschaft bestimmt den Modus, der von Policy Service beim Installieren oder Deinstallieren von Channels verwendet wird. Sie können dafür folgende Werte festlegen:
  • silent: Policy Service Alle Packages werden auf dem Ziel-Tuner im Hintergrundmodus installiert. Dies ist der Standardwert.
  • aspackaged: Policy Service verwendet den im Package angegebenen Installationsmodus.
Wenn Sie möchten, dass die Installation vollständig im Hintergrund ausgeführt wird, sollten Sie die folgenden Tuner-Eigenschaften festlegen, um zu verhindern, dass bei den Clients Statusanzeigen oder Fehlermeldungen angezeigt werden: marimba.tuner.display.noprogress und marimba.tuner.display.noerrors.
Konfiguration bei Fehler erneut anwenden
 
marimba.subscription.reapplyconfigonfail
"true" oder "false"
 
Standard: true
Diese Eigenschaft bestimmt, ob eine zwischengespeicherte Policy erneut angewendet werden soll, wenn Policy Service beim Versuch, eine aktualisierte Policy abzurufen, nicht mit dem Plugin kommunizieren kann.
Bei "true" wendet Policy Service die zwischengespeicherte Policy (die zuletzt heruntergeladene Policy) erneut an, wenn die Kommunikation mit dem Plugin zum Abrufen einer aktualisierten Policy fehlschlägt. Bei "false" versucht Policy Service nicht, die zwischengespeicherte Policy erneut anzuwenden.
(Neu in Version 6.0.)
marimba.subscription.adminusers
<Benutzer1>,
<Benutzer2>,...
Gibt eine durch Kommata getrennte Liste der Administratoren an. Die Liste der Administratoren kann den Namen jedes beliebigen Benutzers enthalten, der sich an den Clientcomputern anmelden kann.
 
Standard: none
Wenn auf dem Computer des Client-Benutzers Policy Service 5.0.1 oder höher installiert ist, können Sie diese Tuner-Eigenschaft festlegen, um die Administratoren anzugeben, die sich temporär auf Computern von Benutzern anmelden können, wenn sie benutzerbasierte Ziele verwenden (z. B. zur Fehlerbehebung). Mit dieser Eigenschaft können Sie sich als Administrator anmelden, ohne dass die Channel des Benutzers gelöscht werden.
Bei der Anmeldung des Administrators geschieht Folgendes:
  • Channel-Subscriptions anderer Benutzer werden nicht entfernt.
  • Wenn Channel vorhanden sind, die einem Administrator zugeordnet sind, werden diese Channel an den Client geliefert.
  • Die für den Administrator festgelegten Eigenschaften werden eingestellt. Möglicherweise werden die für die Endanwender festgelegten Einstellungen durch die des Administrators überschrieben. Wenn die Benutzer sich erneut anmelden und Policy Service eine Aktualisierung durchführt, erhalten die Endanwender ihre Einstellungen wieder.
marimba.subscription.autostart
<Channel_
Name>, “<Argument1>, <Argument2>, <Argument3>“,true oder false
 
Standard: kein Wert
Damit können Sie den angegebenen Channel starten, sobald Policy Service ausgeführt wird.
  • <Channel_Name> gibt den Namen des Channel an, der gestartet werden soll. Diese Angabe ist erforderlich.
  • “<Argument1>,<Argument2>,<Argument3>“
    sind alle Argumente, die dem Channel übergeben werden sollen. Sie sind optional.
  • true oder false gibt an, ob der Channel vor seinem Start aktualisiert werden soll. Dieses letzte Argument ist optional.
 
Beispiel:

Hinweis: Gehen Sie bei der Verwendung dieser Eigenschaft für BMC CM-Channel vorsichtig vor. Einige Channel sind evtl. so konfiguriert, dass sie Policy Service starten und so zu einer Rekursion führen.
marimba.subscription.machinename
eine Zeichenfolge, die den Namen des Computers darstellt.
 
Standard: kein Wert
Damit können Sie den von Policy Service zurückgegebenen Computernamen außer Kraft setzen. Der Name, den Sie hier eingeben, muss mit dem Namen übereinstimmen, der in der flachen Datei des Computers verwendet wird. Standardmäßig ist dieser Eigenschaft kein Wert zugewiesen, und als Computername wird einfach der DNS-Hostname ohne Domäneninformationen verwendet.
marimba.subscription.retryintervalsec
Integer
 
Standard: 30
(30 Sekunden)
Diese Eigenschaft legt das Intervall in Sekunden fest, bevor erneut versucht wird, eine Verbindung mit LDAP herzustellen. Diese Eigenschaft wird auf dem Tuner festgelegt, der als Host für den Transmitter dient, auf dem Policy Service-Plugin veröffentlicht wurde.
marimba.subscription.timeout
Integer
 
Standard: 3600
(3600 Sekunden oder 1 Stunde)
Diese Eigenschaft legt die maximale Zeitdauer in Sekunden fest, innerhalb der Policy Service die Vervollständigung eines Vorgangs abwartet, bevor er fortfährt (wie beispielsweise, dass ein Package eine Mitteilung sendet, dass es einen bestimmten Status erreicht hat).
marimba.subscription.varytime
Integer
 
Standard: 10
(10 Minuten)
Diese Eigenschaft legt die maximale Zeitdauer in Minuten fest, in der geplante Ereignisse (z. B. das Herunterladen oder die Aktualisierung eines Package) aufgeschoben werden können. Dies verbessert die Leistung des Transmitter, da Client-Anfragen bei hoher Auslastung über den gesamten Zeitraum verteilt werden. Wenn beispielsweise diese Eigenschaft auf 10 Minuten festgelegt ist und ein Ereignis um 10:00 Uhr ablaufen soll, kann das Ereignis irgendwann zwischen 10:00 und 10:10 Uhr ablaufen.
marimba.tuner.display.noerrors
true oder false
 
Standard: false
Diese Eigenschaft legt fest, ob der Tuner Fehler- oder Warnmeldungen anzeigen soll oder nicht. Um Fehler- oder Warnmeldungen auszublenden, muss diese Eigenschaft auf true festgelegt werden. Die Fehler- oder Warnmeldungen werden auf eine Systemkonsole ausgegeben. Bitte beachten Sie, dass diese Eigenschaft nicht für Tuner gilt, die ohne Anzeige ausgeführt werden.
marimba.tuner.display.noprogress
true oder false
 
Standard: false
Diese Eigenschaft zeigt an, ob der Tuner eine Statusanzeige anzeigen soll, wenn eine Channel-Subscription eingerichtet wird oder wenn Channel aktualisiert werden. Um die Statusanzeige zu verbergen, setzen Sie diese Eigenschaft auf "true". Bitte beachten Sie, dass diese Eigenschaft nicht für Tuner gilt, die ohne Anzeige ausgeführt werden.

Was sind Package-Eigenschaften?

Package-Eigenschaften stellen Informationen über das Package sowie über dessen Ausführung zur Verfügung. Sie werden auch Channel-Eigenschaften genannt. Diese Informationen werden normalerweise vom Ziel-Tuner verwendet. Packages (dies sind üblicherweise Applikationen, die mit Hilfe von Application Packager in Channel gepackt wurden) verfügen über zusätzliche Eigenschaften, die Channel-Parameter genannt werden. Channel-Parameter stellen Informationen und Einstellungen bereit, die bei der Installation und beim Ausführen der Packages verwendet werden.

Eigenschaften und Parameter von Packages werden in den Dateien properties.txt bzw. parameters.txt gespeichert. Diese Dateien werden normalerweise (im Channel-Verzeichnis) erstellt, wenn der Channel oder das Package erstellt werden. Sie können die Dateien manuell oder mit Hilfe von Channel Copier oder Publisher bearbeiten.

Ein Liste von Package-Eigenschaften und Channel-Parametern finden Sie im Kapitel über Channel-Eigenschaften und -Parameter im BMC CM-Referenzhandbuch (Configuration Management Reference Guide), das auf der BMC Customer Support-Website erhältlich ist.

Eigenschaften und Parameter von Packages werden normalerweise in folgender Form angezeigt:

<Eigenschaftsname>=<Eigenschaftswert>

Eine Anleitung zum Festlegen von Package-Eigenschaften finden Sie unter Festlegen von Package-Eigenschaften.

Eigenschaften zur Steuerung des Neustart von Packages (nur Windows)

Während der Installation von Packages, die mit Application Packager erstellt wurden, können einige Dateien nicht geschrieben werden, weil sie vom Betriebssystem gesperrt wurden. Die Sperrung von Dateien tritt meist auf, wenn eine Datei geöffnet ist oder eine DLL geladen wird. In diesen Fällen wird die neue Datei unter einem temporären Namen auf der Festplatte gespeichert. Sie ist gekennzeichnet, um nach dem Systemstart in den korrekten Namen und Speicherort umbenannt zu werden.

Policy Service installiert während einer Policy-Aktualisierung eventuell mehrere Packages. Wenn in einer Sitzung mehrere Packages installiert werden, schiebt Policy Service den Systemneustart auf, bis alle Packages aktualisiert und installiert sind. Der Neustart kann selbst dann stattfinden, wenn die Installation fehlschlägt, je nachdem, wie weit die Installation fortgeschritten war.

Von Seiten des Policy Service wird das Neustartverhalten von den Tuner-Eigenschaften marimba.subscription.reboot und marimba.subscription.reboot.time gesteuert. Unabhängig von den Einstellungen der einzelnen Packages, überprüft Policy Service die Erforderlichkeit eines Neustarts nur dann, wenn die Eigenschaft marimba.subscription.reboot auf true gesetzt ist.

Policy Service kann die Neustarteigenschaften einzelner Packages berücksichtigen, die mit Hilfe des Application Packager gepackt wurden. Diese Package-Eigenschaften sind in Tabelle 4-8 beschrieben. Zwei dieser Eigenschaften, reboot.allow und reboot.force, bestimmen das Neustartverhalten, wie in Tabelle 4-9 beschrieben. Zwei andere Package-Eigenschaften, reboot.showdialog und reboot.allowcancel, ermöglichen die Steuerung über das Dialogfeld, das angezeigt wird, wenn ein Neustart erforderlich ist.

Eigenschaftsname
Mögliche Werte
Beschreibung
reboot.allow
"true" oder "false"
 
Standard: true
Dieser Parameter zeigt an, dass der Channel den Computer, wenn erforderlich, neu starten kann. Der Standardwert ist "true". Setzen Sie diesen Parameter auf "false", wenn dieses Verhalten nicht erwünscht ist.
Hinweis: Im Hintergrundmodus werden Computer von Channels nicht neu gestartet, es sei denn der Parameter "reboot.force" wird auf "true" gesetzt.
reboot.allowcancel
"true" oder "false"
 
Standard: true
Wenn dieser Parameter auf "true" gesetzt und ein Neustart beim Client erforderlich ist, wird im Neustart-Dialogfeld (nur wenn Benutzeroberfläche im Vordergrund oder teilweise im Hintergrund) eine Neustartoption angezeigt, mit deren Hilfe der Benutzer den Neustart abbrechen kann. Wenn sich das Installationsprogramm im Hintergrundmodus befindet, wird kein Neustart ausgeführt. Dies gilt selbst dann, wenn ein Neustart erforderlich ist.
reboot.force
"true" oder "false"
 
Standard: false
Bei "true" wird nach Abschluss der Installation ein Neustart des Computers erzwungen. Der Standardwert ist "false". Verwenden Sie diesen Parameter vorsichtig, da er zu unerwünschten Benutzerinteraktionen führen kann.
reboot.showdialog
Gültige Werte: "true" oder "false"
 
Standard: true
 
Dieser Parameter zeigt an, ob der Channel im interaktiven Modus (nicht im Hintergrund- oder teilweisen Hintergrundmodus) das Neustart-Dialogfeld anzeigen soll, falls ein Neustart erforderlich ist. Der Standardwert ist "true". Setzen Sie diesen Parameter auf "false", wenn dieses Verhalten nicht erwünscht ist.

Wenn für alle von Policy Service installierten Packages sowohl reboot.allow als auch reboot.force auf "true" gesetzt sind, startet Policy Service das System neu, unabhängig davon, ob ein Neustart erforderlich ist oder nicht.
