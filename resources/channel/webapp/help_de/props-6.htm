Wenn Sie nicht möchten, dass Tuner- oder Package-Eigenschaften weiter auf den Client-Tunern festgelegt werden, können Sie diese mit Policy Manager löschen. Sie löschen Eigenschaften, indem Sie den Wert der Eigenschaft einschließlich der spitzen Klammern (<>) auf <default> setzen.

Wenn Sie eine Tuner-Eigenschaft mit Policy Manager löschen, wird diese aus der Datei prefs.txt im Workspace-Verzeichnis des Tuner gelöscht und hat normalerweise keine Auswirkungen mehr. Hätten Sie dagegen die Eigenschaft bereits beim Packen des Tuner festgelegt, könnte sie über Policy Manager nicht gelöscht werden. Der Grund dafür ist, dass durch Festlegen der Eigenschaft beim Packen die Eigenschaft in der Datei properties.txt im Workspace-Verzeichnis des Tuner (unter Windows gewöhnlich C:\Programme\Marimba\Tuner\.marimba\Marimba\properties.txt) festgelegt wird und Policy Manager die Eigenschaft nicht aus der Datei properties.txt entfernt. Üblicherweise setzt die Datei prefs.txt die in der Datei properties.txt angegebenen Standard-Tuner-Eigenschaften außer Kraft. In diesem Fall ist die Tuner-Eigenschaft jedoch nur in der Datei properties.txt enthalten. Daher wird die Eigenschaft auch angewendet.

Hinweis: Sie können Einstellungen auf Zielebene, wie die Tuner- oder Package-Eigenschaften, nur für jeweils ein Ziel festlegen. Sie können keine Einstellungen auf Zielebene einrichten, wenn Sie sich im Mehrfachauswahlmodus befinden und gleichzeitig Policys für mehrere Ziele bearbeiten. Sie können jedoch für alle Ziele Einstellungen auf Zielebene einrichten, indem Sie das Ziel Alle Clients auswählen.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So löschen Sie Eigenschaften von Zielen:

  1. Klicken Sie auf der Seite "Policy bearbeiten" auf die Registerkarte Erweitert > Tuner/Package-Eigenschaften.

    2. Die Seite "Tuner- und Package-Eigenschaften" wird angezeigt.

  2. Suchen Sie das Textfeld Eigenschaften, und geben Sie hier Name und Wert der Eigenschaft ein, wobei der Wert samt spitzen Klammern (<>) auf <default> gesetzt wird.

    3. Beispiel: marimba.subscription.retrycount=<default>

    Verwenden Sie das unter Tuner- und Package-Eigenschaften, Format beschriebene Format.

  3. Führen Sie zum Festlegen der Package-Eigenschaften in der Tabelle "Package-Eigenschaften" die folgenden Schritte aus:
    1. Wählen Sie aus der Liste Package das Package oder die Gruppe von Packages aus, für die Sie Eigenschaften festlegen möchten. Wählen Sie eine der folgenden Optionen aus:
      • URL-Adresse eines bestimmten Package: Wählen Sie diese Option aus, wenn Sie lediglich die Eigenschaft eines bestimmten Packages festlegen möchten.
      • Alle Packages: Wählen Sie diese Option aus, wenn Sie die Eigenschaft für alle Packages auf dem Tuner (für die Ziele), außer für den Policy Service-Channel, festlegen möchten.
      • Alle Packages mit Subscription: Wählen Sie diese Option aus, wenn Sie die Eigenschaft für alle Packages festlegen möchten, die Teil der Policy sind, außer wenn diese den Status Deinstallieren, Ausschließen, Primär oder Ankündigen besitzen. Auch der Policy Service-Channel ist nicht betroffen.
      • Policy Service Channel: Wählen Sie diese Option aus, wenn Sie die Eigenschaft lediglich für den Policy Service-Channel festlegen möchten.
    2. Geben Sie Name und Wert der Eigenschaft ein. Setzen Sie den Eigenschaftswert dabei auf <default>.
    3. Klicken Sie auf Übernehmen.

      4. Die Eigenschaft wird dem Textfeld "Eigenschaften" hinzugefügt.

      Hinweis: Sie können die URL-Adresse des Package sowie den Namen und den Wert der Package-Eigenschaft auch direkt in das Textfeld Eigenschaften eingeben. Verwenden Sie das unter Tuner- und Package-Eigenschaften, Format beschriebene Format.

  4. Überprüfen Sie die im Textfeld Eigenschaften aufgelisteten Tuner- und Package-Eigenschaften.

    5. Hinweis: Durch Löschen einer Eigenschaft aus dem Textfeld Eigenschaften wird diese nicht von den Zielclients entfernt. Daher bleibt die Eigenschaft auf dem Ziel-Tuner bestehen (nicht von Policy Manager verwaltet). Zum Entfernen einer Tuner- oder Package-Eigenschaft von den Zielclients setzen Sie den Wert auf <default>.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.
