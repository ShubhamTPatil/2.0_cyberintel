Bei Nichtbeachtung der Best Practices kann es zu Konflikten bei Tuner- und Channel-Eigenschaften kommen, wenn verschiedene Werte für die gleiche Eigenschaft in unterschiedlichen Gruppen festgelegt werden. In diesem Abschnitt wird beschrieben, wie Konflikte bei Eigenschaftswerten behoben werden. Weitere Informationen zu Tuner- und Package-Eigenschaften finden Sie unter Einstellen der Tuner- und Package-Eigenschaften für ein Ziel.

Tuner-Eigenschaften. Wenn Sie einen Client in mehr als eine Gruppe einbinden, können Sie theoretisch für eine einzelne Tuner-Eigenschaft in der Policy jeder Gruppe unterschiedliche Werte festlegen. Da Konflikte bei den Tuner-Eigenschaften nicht mit einem Vorrangsschema gelöst werden können, ist keine Voraussage darüber möglich, welcher Wert sich durchsetzt.

BMC empfiehlt, die Tuner-Eigenschaften einheitlich für alle Gruppen festzulegen, in denen ein Client Mitglied ist.

Package-Eigenschaften. Wenn eine Package-Eigenschaft sowohl für alle Packages mit Subscription als auch für ein einzelnes Package angegeben ist, hat der Eigenschaftswert für das einzelne Package Vorrang. Stellen Sie sich beispielsweise folgende Eigenschaftseinstellungen vor:

semisilent=false für alle Packages mit Subscription
(Die Syntax lautet semisilent,subscribers=false.)

semisilent=true für das Package „WordProcessor App“
(Die Syntax lautet semisilent,http://trans.company.com:5282/Applications/WordProcessorApp=true.)

In diesem Fall wird der Wert der Eigenschaft semisilent für das Package "WordProcessorApp" auf true gesetzt. Wenn die Eigenschaft semisilent für andere Subscription-Packages Null ist, legt Policy Manager den Wert auf false fest.
