In diesem Abschnitt wird beschrieben, wie der Aktualisierungszeitplan für Policy Service festgelegt wird. Wenn Sie keinen Zeitplan festlegen, wird gemäß der Standardeinstellung alle 90 Minuten eine Aktualisierung durchgeführt.

Wenn Sie für das Ziel eine Blackout-Periode festgelegt haben, können Sie Policy Service davon ausnehmen. Die Ausnahme ist dann nützlich, wenn Sie bei der Aktualisierung oder Installation von Packages während einer Blackout-Periode Notfallsituationen erwarten. Weitere Informationen dazu finden Sie unter Festlegen der Blackout-Periode für ein Ziel.

Wenn mehrere Benutzer die Seite "Zeitplan" von Policy Service für das gleiche Ziel bearbeiten, werden die zuletzt gespeicherten Änderungen übernommen.

Hinweis: Sie können Einstellungen auf Zielebene, wie zum Beispiel den Aktualisierungszeitplan für Policy Service, nur für jeweils ein Ziel festlegen. Sie können keine Einstellungen auf Zielebene einrichten, wenn Sie sich im Mehrfachauswahlmodus befinden und gleichzeitig Policys für mehrere Ziele bearbeiten. Sie können jedoch für alle Ziele Einstellungen auf Zielebene einrichten, indem Sie das Ziel Alle Clients auswählen.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite "Policy bearbeiten" auf der Registerkarte "Packages" befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So legen Sie den Aktualisierungszeitplan für Policy Service fest:

  1. Klicken Sie auf der Seite "Policy bearbeiten" auf die Registerkarte Policy Service-Zeitplan.
  2. Wählen Sie Policy Manager zum Festlegen des Aktualisierungszeitplans für Policy Service verwenden aus.

    3. Wenn Sie diese Option nicht auswählen, können Sie keinen Zeitplan festlegen. Policy Service folgt in diesem Fall dem Standard-Aktualisierungszeitplan (alle 90 Minuten).

  3. Geben Sie im Abschnitt Wiederholung an, wie häufig Policy Service aktualisiert werden soll:
    • Täglich: Policy Service wird an jedem Wochentag (Montag bis Freitag) oder alle 1 bis 365 Tage aktualisiert, je nachdem, welchen Wert Sie festlegen.
    • Wöchentlich: Policy Service kann alle 1 bis 52 Wochen aktualisiert werden, je nachdem, welchen Wert Sie festlegen. Dabei wird die Aktualisierung in der festgelegten Woche nur an dem ausgewählten Tag durchgeführt. Sie müssen mindestens einen Wochentag auswählen.
    • Monatlich: Policy Service kann alle 1 bis 12 Monate an dem festgelegten Tag aktualisiert werden. Wenn Sie den 30. oder 31. wählen und der Monat der Aktualisierung keinen solchen Tag hat (wie Februar), wird die Aktualisierung am ersten Tag des Folgemonats durchgeführt.
  4. Geben Sie im Abschnitt Aktualisierungszeit die Uhrzeit an, zu der Policy Service aktualisiert werden soll.
    • Aktual. um: Legen Sie die Uhrzeit fest, zu der die Aktualisierung erfolgen soll.
    • Aktual. alle: Legen Sie fest, wie häufig am Tag eine Aktualisierung erfolgen soll. Sie können festlegen, dass Aktualisierungen in einem bestimmten Intervall von Minuten oder Stunden am Tag der Aktualisierung stattfinden sollen. Sie können die Aktualisierung aber auch für einen bestimmten Zeitraum innerhalb des Tages festlegen. Aktualisierungen können beispielsweise jede Stunde zwischen 9:00 und 17:00 Uhr erfolgen oder alle 30 Minuten zwischen 17:00 und 9:00 Uhr.

Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.

Hinweis: Wenn Sie den Policy Service-Aktualisierungszeitplan löschen, weil Sie eine Policy für ein Ziel gelöscht haben, wird Policy Service auch weiterhin gemäß dem festgelegten Zeitplan aktualisiert. Die Standardeinstellung für den Policy Service-Aktualisierungszeitplan wird nicht wiederhergestellt. Der Policy Service verwendet den festgelegten Aktualisierungszeitplan so lange, bis Sie einen neuen Policy Service-Aktualisierungszeitplan festgelegt haben.
