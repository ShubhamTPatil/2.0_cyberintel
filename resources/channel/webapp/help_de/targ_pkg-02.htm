Folgende Zieltypen stehen zur Auswahl:

    • Benutzer (Client mit Anmeldename)
    • Computer
    • Benutzergruppe
    • Computergruppe
    • Container oder Organisationseinheit (OU)
    • Domäne in Active Directory
    • Collection, die das Ergebnis einer Datenbank- oder LDAP-Abfrage ist. Weitere Informationen dazu finden Sie unter Was ist eine Collection?.
    • Alle Clients, eine spezielle virtuelle Gruppe, die alle Clients umfasst. Weitere Informationen dazu finden Sie unter Was ist das Ziel "Alle Clients"?.

Nachdem Sie einem Ziel ein Package zugewiesen haben, können Sie jedem Package einen Installationsstatus sowie Zeitpläne und Eigenschaftseinstellungen zuweisen, die sich auf die Installation bei den einzelnen Clients auswirken. Alle Informationen, die Sie einem Ziel zuweisen, werden als Policy bezeichnet.

Hierarchische Auflösung von Policys. Die Auflösung von Policys für Targets erfolgt hierarchisch. Zum Beispiel ist folgender Client gegeben:

cn=user1,cn=computers,ou=finance,dc=company,dc=com

Der Client erhält Policys, die den folgenden Zielen zugewiesen sind:

    • cn=computer,ou=finance,dc=company,dc=com
    • ou=finance,dc=company,dc=com
    • dc=company,dc=com
