Profile enthalten Konfigurationseinstellungen für BMC CM-Infrastrukturkomponenten, jedoch keine Produktbinärdateien. Sie können mehrere Profile erstellen, jeweils ein Profil für die verschiedenen Arten von Komponenten, z. B. Transmitter, Mirror, Repeater, Proxies und verwaltete Clients. Jedes Profil wird als ein Segment des Infrastructure Service gespeichert. Wenn Sie eine Profileinstellung ändern, erhält ein Client bei der nächsten Aktualisierung seines Infrastructure Service-Channel die neue Profileinstellung.

Wenn Sie ein Profil auswählen und über Policy Manager einem Client zuweisen, wird die Tuner-Eigenschaft marimba.tuner.update.profile auf den Namen des angegebenen Profils gesetzt.

Weitere Informationen zum Erstellen von Profilen und Profiltypen finden Sie in der Hilfe zu Setup und Deployment.
