Wenn Sie einem Ziel ein anderes Profil zuweisen (oder einem Ziel ohne Profil ein solches zuweisen) möchten, können Sie diese Änderung in der Policy des Ziels vornehmen. Weitere Informationen zum Erstellen und Bearbeiten von Profilen und Profiltypen finden Sie in der Hilfe zu Setup und Deployment.

Beachten Sie beim Ändern des Profils auf Zielen die folgenden Punkte:

    • Zielen wird ein neues Profil zugewiesen, indem Sie in Policy Manager die Registerkarte „Profile“ verwenden oder die Tuner-Eigenschaft marimba.tuner.update.profile in Tuner Administrator festlegen. Dieses Dokument enthält eine Anleitung zum Verwenden von Policy Manager für die Zuweisung eines neuen Profils. Wenn Sie Policy Manager zum Zuweisen eines Profils verwenden, können Sie Collections nutzen. Eine Collection ist eine Liste von Benutzern oder Computern, die bestimmten Kriterien entsprechen, die Sie in einer LDAP- oder Datenbankabfrage angeben. Nach Ausführen der Abfrage können Sie der daraus resultierenden Liste (der Collection) mit Policy Manager ein Profil zuweisen.
    • Der Infrastructure Service-Channel wird nach einem Zeitplan ausgeführt. Beim nächsten Start des Infrastructure Service auf einem Computer werden alle verfügbaren Änderungen an Profileinstellungen und Tuner-Binärdateien heruntergeladen und übernommen. Einige Änderungen werden auf die Computer erst angewendet, nachdem der Tuner auf dem Computer neu gestartet wurde. Informationen zum Aktualisieren der Binärdateien des Tuner finden Sie im BMC CM-Aktualisierungshandbuch (BMC Configuration Management Upgrade Guide), das auf der BMC Customer Support-Website erhältlich ist.

Vor dem Zuweisen der Policy müssen Sie folgende Tasks ausführen:

    • Wenn das Profil noch nicht vorhanden ist, erstellen Sie das neue Profil, das Sie dem Ziel zuweisen möchten.
    • Wenn Sie das Profil einer Collection zuweisen möchten, definieren Sie eine Collection, die den Satz von Computern enthält, denen Sie das neue Profil zuweisen möchten:
    • Erstellen Sie eine Abfrage, und geben Sie die Kriterien für den gewünschten Satz von Computern an. Weitere Informationen dazu finden Sie unter Was ist eine Collection?.
    • Führen Sie die Abfrage entweder durch Festlegen eines Zeitplans oder manuell aus.

      • Nach Ausführung der Abfrage wird die daraus resultierende Liste der Computer (die Collection) in Policy Manager als Ziel angezeigt.

      Hinweis: Sie können Einstellungen auf Zielebene wie das Profil nur für jeweils ein Ziel einrichten. Sie können keine Einstellungen auf Zielebene einrichten, wenn Sie sich im Mehrfachauswahlmodus befinden und gleichzeitig Policys für mehrere Ziele bearbeiten. Sie können jedoch für alle Ziele Einstellungen auf Zielebene einrichten, indem Sie das Ziel Alle Clients auswählen.

Bei diesem Vorgang wird davon ausgegangen, dass Sie sich auf der Seite „Policy bearbeiten“ auf der Registerkarte „Packages“ befinden. Weitere Informationen dazu finden Sie unter Allgemeine Anweisungen zum Erstellen und Bearbeiten von Policys.

So ändern Sie das Profil für ein Ziel:

  1. Klicken Sie auf der Seite „Policy bearbeiten“ für das Ziel auf Erweitert > Profil.

    2. Wenn dem Ziel bereits ein Profil zugewiesen ist, werden der Name des Profils und die URL-Adresse des Infrastructure Service-Channel (die Quelle des Profils) auf dieser Registerkarte angezeigt. Wenn dem Ziel kein Profil zugewiesen ist, wird kein Profilname angezeigt.

  2. Klicken Sie auf der Registerkarte „Profil“ auf Bearbeiten.

    3. Die Seite „Profil auswählen“ wird angezeigt. Wenn dem Ziel kein Profil zugewiesen ist, zeigt Policy Manager die Profile im ersten Infrastructure Service-Channel auf dem Transmitter an, von dem der Policy Manager-Channel per Subscription bezogen wird. Beispiel: Wenn eine Subscription des Policy Manager-Channel von http://trans.company.com:5282/Marimba/Current/SubscriptionManager eingerichtet ist, werden die Profile im ersten Infrastructure Service-Channel angezeigt, der sich auf demselben Transmitter befindet, z. B. http://trans.company.com:5282/Marimba/Current/InfrastructureService.

  3. Um die Profile eines anderen Standorts anzuzeigen, geben Sie die URL-Adresse des Transmitter (z. B. http://trans.company.com:5282) und, falls Zugriffsberechtigungen für den Transmitter festgelegt sind, den Benutzernamen und das Passwort für Channel-Subscriptions ein. Klicken Sie auf Start.

    4. Im linken Bereich der Seite wird eine Liste der Profile angezeigt. Diese Profile stehen im Infrastructure Service auf dem angegebenen Transmitter zur Verfügung. Jedes Profil wird als ein Segment des Infrastructure Service-Channel gespeichert. Welche Profile in der Profilliste angezeigt werden, ist daher vom angezeigten Infrastructure Service abhängig.

  4. Klicken Sie auf ein Profil, so dass dieses im rechten Bereich der Seite unter Ausgewähltes Profil angezeigt wird, und klicken Sie dann auf OK.

    5. Sie kehren zur Registerkarte „Profil“ zurück, auf der das ausgewählte Profil angezeigt wird.

  5. Nachdem Sie die Policy bearbeitet haben, sehen Sie sich die Vorschau an, und speichern Sie die Policy. Weitere Informationen dazu finden Sie unter Erstellen der Vorschau und Speichern von Policy-Änderungen.

Durch dieses Verfahren wird folgende Tuner-Eigenschaft für Zielcomputer eingestellt:

marimba.tuner.update.profile

Der Wert dieser Eigenschaft ist der Name des neuen Profilsegments, das Sie den Zielcomputern zuweisen möchten. Der Name eines Profilsegments hat folgendes Format: .profile_<Name>

Beispiel: marimba.tuner.update.profile=.profile_ny_mirrors

Beim nächsten Ausführen von Policy Service auf einem Computer wird diese Tuner-Eigenschaft in der Datei prefs.txt des Tuners festgelegt. Beim nächsten Ausführen von Infrastructure Service auf dem Computer, und nachdem der Tuner neu gestartet wurde, wird das neue Profil auf diesen Computer angewendet.
