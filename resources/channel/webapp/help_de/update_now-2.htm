Informationen zur Installation der in diesem Abschnitt erwähnten Komponenten finden Sie im BMC CM Installations- und Deployment-Handbuch (BMC Configuration Management Installation and Deployment Guide). Informationen zur Konfiguration von Deployment Manager und Deployment Service finden Sie im Administratorhandbuch für Server Management (Server Management Administrator's Guide), Informationen zur Konfiguration von Report Center und Scanner Service im Administratorhandbuch für Report Center (Report Center Administrator's Guide). Diese Dokumente sind auch auf der BMC Customer Support-Website erhältlich.

Serverseitig. Folgende Komponenten müssen auf dem gleichen Computer wie Policy Manager installiert und konfiguriert sein:

    • Report Center
    • Deployment Manager

      • Hinweis: BMC empfiehlt, Deployment Manager auf einem anderen Computer als Policy Manager auszuführen.

Bei einer Neuinstallation von Deployment Manager müssen Sie sich anmelden und der Lizenzvereinbarung zustimmen, bevor Sie diesen mit Policy Manager verwenden können. Außerdem müssen Sie die Deployment Manager-Einstellungen für die Konsole konfigurieren (Systemeinstellungen > Allgemein > Deployment Manager-Integrationseinstellungen), so dass Policy Manager korrekt mit Deployment Manager kommunizieren kann. Folgende Informationen werden benötigt:

    • Hostname des Computers, auf dem Deployment Manager ausgeführt wird. Dieser Computer kann der gleiche sein, auf dem Policy Manager ausgeführt wird, oder es kann ein anderer Computer sein, auf dem Deployment Manager ausgeführt wird.
    • Portnummer, die vom Tuner verwendet wird, auf dem der Deployment Manager-Channel ausgeführt wird. Der Standardwert ist 7717.
    • Benutzername und Passwort für den Zugriff auf Deployment Manager. Deployment Manager muss für den Zugriff dieses Benutzers konfiguriert sein.
    • Benutzername und Passwort für die Remote-Administration der Client-Tuner, zu denen Deployment Manager eine Verbindung herstellt. Der Benutzername und das Passwort wurden den Client-Tunern entweder beim Erstellen der Tuner-Installationsprogramme oder beim Herstellen einer Verbindung mit den Tunern über Tuner Administrator zugewiesen. Alle Client-Tuner müssen den gleichen Benutzernamen und das gleiche Passwort für die Remote-Administration verwenden.

Auf der Seite „Deployment Manager-Integrationseinstellungen“ können Sie weitere, optionale Einstellungen konfigurieren. Darüber hinaus können Sie sich bei Deployment Manager anmelden, um Standardeinstellungen zu ändern, wie z. B. den Statusport (der Port, der für das Senden von Statusmeldungen von den Zielservern zurück an Deployment Manager verwendet wird). Weitere Informationen zu den Einstellungen von Deployment Manager finden Sie in der Online-Hilfe zu Deployment Manager.

Bevor Sie Aktualisierungen durchführen können, müssen Sie auch die Funktion für sofortige Policy-Aktualisierung aktivieren, die standardmäßig deaktiviert ist. Weitere Informationen dazu finden Sie unter Aktivieren und Deaktivieren von sofortigen Policy-Aktualisierungen.

Clientseitig. Folgende Komponenten müssen auf den gleichen Clients wie Policy Service installiert und konfiguriert sein:

    • Deployment Service
    • Scanner Service

      • Hinweis: Wenn Sie die Funktion zur sofortigen Policy-Aktualisierung für Collections verwenden möchten, muss Scanner Service nach dem Erstellen der Collections mindestens einmal auf den Clients ausgeführt werden. Durch das erneute Ausführen von Scanner Service auf diesen Clients wird sichergestellt, dass die für die sofortige Policy-Aktualisierung erforderlichen Informationen in der Datenbank verfügbar sind.

      Weitere Information zur Konfiguration von BMC CM-Produkten und Firewalls finden Sie im BMC CM Installations- und Deployment-Handbuch (BMC Configuration Management Installation and Deployment Guide) und im BMC CM Infrastruktur-Handbuch (BMC CM Infrastructure Guide), die auf der BMC Customer Support-Website erhältlich sind.
