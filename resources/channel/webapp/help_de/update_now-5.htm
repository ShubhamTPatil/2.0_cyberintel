Wenn Sie eine Policy-Aktualisierung für ein Ziel gestartet haben, wird die Seite „Status der Policy-Aktualisierung“ angezeigt, auf der Sie den Status der zum Ziel gehörenden Computer überwachen können.

Wenn Sie nach dem Starten einer Policy-Aktualisierung zur Seite "Zielansicht" bzw. "Zielinformationen" zurückkehren, wird neben der Schaltfläche Aktualisieren das Symbol für die laufende Aktualisierung angezeigt. Dadurch wird angezeigt, dass die Policy-Aktualisierung für dieses Ziel gerade ausgeführt wird. Wenn Sie während einer Policy-Aktualisierung auf die Seite "Zielansicht" oder "Zielinformationen" für ein anderes Ziel wechseln, wird neben der Schaltfläche Aktualisieren eine abgeblendete Version des Symbols angezeigt. Damit wird angegeben, dass Sie keine weitere Policy-Aktualisierung starten können (auch nicht für ein anderes Ziel), da gegenwärtig bereits eine solche ausgeführt wird. Durch Klicken auf Aktualisieren wird die zusätzliche Option Status eines anderen Ziels anzeigen unten in der Dropdownliste angezeigt.

Nach Ausführung einer Policy-Aktualisierung kann der Status der letzten Ausführung auf der Seite „Zielansicht“ oder der Seite „Zielinformationen“ angezeigt werden. Durch Klicken auf Aktualisieren wird die zusätzliche Option Status der letzten Aktualisierung anzeigen unten in der Dropdownliste angezeigt. Weitere Informationen dazu finden Sie unter Überwachen und Anzeigen des Status einer Policy-Aktualisierung. Policy Manager speichert kein Verlaufsprotokoll für ausgeführte Policy-Aktualisierungen.

Auf der Seite „Status der Policy-Aktualisierung“ können Sie Informationen zur Policy-Aktualisierung anzeigen, und zwar zu folgenden Punkten:

    • Ziel. Name des Ziels, das Sie für die Policy-Aktualisierung ausgewählt haben.
    • Gestartet am. Datum und Uhrzeit, zu der Policy Manager eine Verbindung mit den zum Ziel gehörenden Clients hergestellt und die angegebene Policy-Aktualisierung gestartet hat.
    • Verbindungsgröße. Die Gesamtzahl der Clients, mit denen Policy Manager eine Verbindung herzustellen versucht, um die angegebene Policy-Aktualisierung auszuführen.
    • Fortschritt. Der Stand der Policy-Aktualisierung. Dabei handelt es sich um den Prozentsatz der Clients, mit denen Policy Manager eine Verbindung hergestellt hat und auf denen die angegebene Policy-Aktualisierung gestartet wurde.

      • Hinweis: Der Fortschritt der Policy-Aktualisierung gibt an, ob Policy Manager eine Verbindung zu den Clients herstellen und die Policy-Aktualisierung starten konnte. Es wird nicht angegeben, ob die angegebene Aktualisierung auf den Clients erfolgreich war.

Sie können außerdem den aktuellen Status der einzelnen Clients (Tuner) anzeigen, auf denen Sie die Policy-Aktualisierung ausführen. Die Clients können einen der folgenden Status aufweisen:

    • Ausstehend. Gibt an, dass Policy Manager derzeit versucht oder demnächst versuchen wird, eine Verbindung herzustellen und die Policy-Aktualisierung auf dem Client zu starten.
    • Angehalten. Gibt an, dass Sie oder ein anderer Administrator die Policy-Aktualisierung über Policy Manager angehalten haben. Dieser Status wird evtl. auch angezeigt, wenn die Policy-Aktualisierung angehalten wurde, weil der Mindestwert nicht erreicht wurde. („Mindestwert“ ist ein Wert von Deployment Manager.) Der Standardmindestwert ist 0. Das bedeutet, dass das Deployment für alle Clients in der Gruppe fortgesetzt wird, unabhängig davon, auf wie vielen Clients es bereits fehlgeschlagen ist oder erfolgreich war.
    • Erfolgreich. Gibt an, dass Policy Manager eine Verbindung mit dem Client hergestellt und die Ausführung gestartet hat.
    • Fehlgeschlagen. Gibt an, dass Policy Manager eine Verbindung mit dem Tuner hergestellt hat, die Policy-Aktualisierung auf dem Client jedoch nicht abgeschlossen werden konnte.

Durch Klicken auf Erfolgreich oder Fehlgeschlagen können Sie die Logs für einen bestimmten Computer anzeigen.
