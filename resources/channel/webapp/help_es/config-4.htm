Normalmente, los campos del servicio de directorios de la página Configuración de complemento ya están rellenados con información del servicio de directorios especificado para CMS. En función del servicio de directorios que utilice, es posible que varíe la lista de campos que se deben rellenar. La página Configuración de complemento incluye los siguientes campos del servicio de directorios:

    • Nombre de host y puerto: el nombre de host y el puerto del servicio de directorios con el formato <nombrehost>:<puerto>. El servicio de directorios que especifique aquí se utilizará para almacenar políticas. Las entradas predeterminadas son las del servicio de directorios especificado para CMS, el cual se utiliza para la autenticación de inicio de sesión. (Normalmente no es necesario rellenar este campo para Active Directory.)

      • Si su organización dispone de servicios de directorios replicados para almacenar políticas, utilice una asignación de servicio de directorios a repetidor. Puede aprovechar los servicios de directorios replicados si configura Policy Management de modo que cada repetidor establezca contacto con un servicio de directorios cercano y se elimine la necesidad de establecer contacto con el asignado al transmitter principal. Además, se puede asignar una lista de servicios de directorios a cada repetidor. Si un servicio de directorios de la lista falla, el repetidor intenta establecer contacto con el siguiente, con lo que se evitan los problemas relacionados con puntos erróneos únicos. Para obtener información sobre cómo asignar transmitters a servicios de directorios, consulte la Guía del administrador de Policy Management.

    • Usar una conexión SSL entre el transmitter y el servicio de directorios: marque esta casilla para activar Secure Sockets Layer (SSL) y conseguir una comunicación segura entre el transmitter y el servicio de directorios. Para obtener más información, consulte la sección sobre seguridad y SSL de Guía del administrador de infraestructuras de BMC CM que encontrará en el sitio web de soporte al cliente de BMC.
    • DN base: nombre distinguido de la base para la conexión del servicio de directorios. DN base define el alcance de la vista de directorios desde el punto de vista del complemento. En la mayoría de los casos, esto equivale al sufijo Sun ONE Directory. Normalmente no es necesario rellenar este campo para Active Directory.

      • Por ejemplo , en ADAM o Sun ONE Directory, puede utilizar dc=compañía,dc=com.

    • DN bind: nombre distinguido de la cuenta de usuario que el Complemento Policy Service utiliza cuando establece una conexión con el servicio de directorios. Esta cuenta de usuario debe contar con permisos de lectura en el ámbito del directorio definido por la entrada DN base. Este ámbito normalmente se asigna a un árbol de directorios entero.

      • Para Active Directory, utilice uno de los siguientes formatos:

    • El nombre distinguido completo (DN), como por ejemplo cn=Administrador,cn=Usuarios,dc=compañía,dc=com
    • El nombre principal de usuario (UPN), como Administrador@empresa.com

      • Para ADAM, utilice el nombre distinguido completo (DN), como por ejemplo cn=Administrador,cn=Usuarios,dc=compañía,dc=com

      Para Sun ONE Directory, utilice uno de los siguientes formatos:

    • El nombre distinguido completo (DN), como por ejemplo uid=Administrador,ou=General,dc=compañía,dc=com
    • El nombre común (CN) del administrador de directorios , como por ejemplo cn=Administrador de directorios
    • Bind DN password: la contraseña necesaria para acceder al servicio de directorios. El complemento utiliza esta contraseña para la conexión del servicio de directorios.
    • Tamaño del grupo de conexiones del servicio de directorios: el Complemento Policy Service utiliza el mecanismo de agrupación para establecer y mantener conexiones al servicio de directorios. Este parámetro define el número máximo de conexiones establecidas en la agrupación (normalmente se puede utilizar el valor predeterminado, 25).
    • El tiempo de caducidad (en minutos) de la última conexión de host correcta: si especifica una lista de nombres de host para conmutación en caso de error de servicios de directorio, Policy Manager recorre la lista hasta que consigue conectarse correctamente a un host. Policy Manager utiliza la conexión al host establecida hasta que transcurra el tiempo de caducidad que haya especificado. A continuación, intenta realizar una nueva conexión con el primer nombre de host de la lista.
    • Permitir usar las políticas como modelos para máquinas aprovisionadas recientemente: si va a integrar la instalación del tuner con el aprovisionamiento de SO, al seleccionar esta casilla de verificación se identifican los grupos del servicio de directorios cuya política desea utilizar como modelo para instalar aplicaciones y contenido en las máquinas aprovisionadas recientemente. Para obtener más información, consulte el capítulo dedicado a la integración de la instalación del tuner con el aprovisionamiento de sistema operativo en la BMC CM Deployment Guide, que puede obtenerse en el sitio web de soporte al cliente de BMC.
