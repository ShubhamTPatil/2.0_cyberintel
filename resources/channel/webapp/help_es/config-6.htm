Decida si desea utilizar o no la función ACL durante el proceso de instalación y configuración. Para determinar si la función ACL está activada o desactivada, coloque el puntero del ratón sobre el icono de estado que se encuentra en el ángulo superior derecho de la ventana de la consola.

Nota: BMC recomienda asignar las ACL y los permisos a los administradores antes de activar la función ACL. De lo contrario, los administradores que inicien la sesión en Policy Manager no podrán administrar las políticas. Se recomienda como buena práctica asignar las ACL y los permisos a los administradores, formarlos, y después activar la funcionalidad ACL Policy Manager cuando esté preparado para aplicar los permisos.

Antes de utilizar Policy Manager para activar la función de control de acceso, utilice la consola para realizar las siguientes configuraciones:

    • Autenticación de usuario
    • Sincronización de LDAP
    • Listas de control de acceso
    • Asignar permisos de las listas de control de acceso
    • Asignar permisos para aplicaciones

Para obtener más información, consulte la ayuda de la consola relativa a la configuración del sistema.

Activar y desactivar la funcionalidad ACL

Una vez configuradas las listas de control de acceso, los administradores principales pueden utilizar Policy Manager para activar la funcionalidad ACL. Las ACL están desactivadas de forma predeterminada.

Puede activar o desactivar la funcionalidad ACL utilizando la ficha Configuración o la interfaz de línea de comandos.

Para activar la funcionalidad ACL utilizando las opciones de configuración

    • Haga clic en la ficha Configuración.
    • Haga clic en Opciones de ACL.

      • Aparecerá la página de activación del control de acceso.

    • Seleccione la casilla de verificación Activar/desactivar control de acceso.
    • Haga clic en Aceptar.

Para activar la función ACL utilizando la línea de comandos

    • En la línea de comandos, introduzca el comando siguiente:

      • runchannel <Policy_Manager_URL> -user <nombre_usuario> -password <contraseña> -configSet marimba.subscription.acl true

      Puede desactivar la función ACL con el mismo comando, sustituyendo true por false. También puede utilizar la opción -preview para obtener una vista previa del cambio que desee realizar.

      Asegúrese de reiniciar Policy Manager después de activar la función ACL.
