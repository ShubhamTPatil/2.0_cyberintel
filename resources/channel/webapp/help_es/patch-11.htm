Este tema se incluye como referencia para ayudarle a llevar a cabo el procedimiento descrito en Configurar parámetros de reinicio de instalaciones de parches. La configuración de reinicio afecta a las siguientes propiedades de tuner:

    • marimba.reboot.interact: especifica si se mostrará o no un mensaje de aviso advirtiendo a los usuarios acerca del reinicio. El valor predeterminado es verdadero (se mostrará un mensaje de aviso).
    • marimba.reboot.interact.allowSnooze: especifica si se permite a los usuarios posponer el reinicio. Si se establece como verdadero, aparecerá un cuadro de diálogo en el que los usuarios podrán posponer el reinicio. Si se establece como falso, aparecerá un cuadro de diálogo de aviso en el que se mostrará el temporizador de cuenta atrás para el reinicio. El valor predeterminado es verdadero.
    • marimba.reboot.interact.snooze.maxTime: especifica el tiempo máximo (en milisegundos) que los usuarios podrán posponer el reinicio. El valor predeterminado es 0 (no existe un tiempo máximo).
    • marimba.reboot.interact.timer: especifica el tiempo de espera o de cuenta atrás en segundos para el reinicio. Este valor es la hora de inicio que se mostrará en el cuadro de diálogo de aviso en caso de que establezca marimba.reboot.interact.allowSnooze como falso. El valor predeterminado es 60 segundos. Si establece este valor como 0, la máquina se reiniciará inmediatamente.
    • marimba.reboot.never: especifica si se reiniciará o no la máquina automáticamente en caso de que sea necesario (por ejemplo, tras la instalación de parches). Establezca el valor como verdadero si no desea que la máquina se reinicie automáticamente.

Puede acceder a estas propiedades en la ficha Avanzado > Propiedades de Tuner/paquete cuando esté editando una política.
