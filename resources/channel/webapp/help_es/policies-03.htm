Si instala la solución BMC Change and Configuration Management (CCM), que incorpora las funcionalidades de alto nivel de BMC Business Service Management (administración de servicios empresariales, BSM), podrá ejecutar y trabajar con tareas de políticas desde el sistema Action Request (AR) o desde el sistema Configuration Management (CM). Policy Manager le permite ejecutar y controlar los cambios de políticas que se originan en el sistema AR relacionando tareas de políticas con los ID de cambios principales. BMC CM también recoge y mantiene un registro de uso que podrá visualizar en Report Center.

El sistema AR rellena automáticamente los campos Cambiar ID e ID de tarea en la página Editar política.

Nota: Si utiliza una solución Change Management de otras empresas, podrá especificar también manualmente los ID de cambio y de tareas e introducir la información de cambios y tareas en archivos de log.

El sistema AR rellena también la información que se indica a continuación cuando se asignan tareas relacionadas con cambios en las políticas de BMC Configuration Management:

    • Información de destinos disponible a través de relaciones LDAP en CMDB (base de datos de administración de configuraciones)
    • Paquetes relacionados como Elementos de la Librería de software (SLIs) dentro de la Librería de software definitivo (DSL)

Para obtener más información sobre la solución CCM, consulte el apéndice Descripción general de CCM en la Guía del administrador de Policy Management que se encuentra disponible en el sitio web de soporte al cliente de BMC. Para obtener información sobre la configuración de Task Management System Integration, consulte la BMC Remedy 7.0 Task Management Administrator’s Guide en el sitio web de soporte al cliente de BMC.
