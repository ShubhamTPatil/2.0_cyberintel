Los estados de instalación determinan de qué modo se instalará, y si se instalará o no, un paquete en un destino. Para cada paquete asignado a un destino, puede establecer un estado principal. En Policy Manager, puede asignar estados de instalación a un paquete que forme parte de la política de uno o más clientes finales.

La siguiente tabla describe los diferentes estados disponibles para los paquetes. Los estados principales descargar y anunciar le permiten establecer un estado secundario para controlar el modo en que se instalará el paquete. Puede asociar una planificación de activación al estado secundario para especificar un periodo de tiempo en el futuro durante el cual deberá llevarse a cabo la instalación en todos los equipos de la empresa, incluidos aquellos clientes finales que se conectan a la red en raras ocasiones. El estado secundario predeterminado es instalar.

Instalar
Descarga e instala el paquete en modo silencioso. (En una versión anterior, este estado se llamaba suscribir.)

Instalar-iniciar
Instala el paquete y ejecuta a continuación el software instalado. (En una versión anterior, este estado se llamaba suscribir e iniciar.) El paquete se inicia cada vez que se ejecuta Policy Service. Si el paquete ya está ejecutándose, Policy Service no se reiniciará.
Desinstalar
Instalar-persistir
Instala el paquete y no permite eliminarlo, aun cuando se haya eliminado el cliente final del grupo de destino. Este estado es muy útil para instalar actualizaciones de software del sistema. El paquete puede eliminarse incluso cuando esté habilitado el estado desinstalar. (En una versión anterior, este estado se llamaba suscribir-persistir.)
N/D
Instalar-iniciar-persistir
Instala el paquete y no permite eliminarlo aunque se haya eliminado el cliente final del grupo de destino, al igual que sucede en el estado instalar-persistir. Inicia el software instalado tras la operación de instalación. El paquete se inicia cada vez que se ejecuta Policy Service. Si el paquete ya está ejecutándose, Policy Service no se reiniciará.
N/D
Excluir
Excluye el paquete del cliente final aunque éste sea miembro de un grupo suscrito. Este estado es muy útil cuando desea suscribir todos los miembros de un grupo muy amplio de un paquete excepto dos de ellos. Este estado tiene la prioridad más alta. (Para obtener más información, consulte Prioridad de estados.)
Nota: para utilizar el estado excluir, debe crear una política en la que se especifiquen los clientes finales que desea excluir (de la recepción del paquete), además de la política que haya suscrito inicialmente el grupo más amplio del paquete.
N/D
Desinstalar
Elimina el paquete. Si el software instalado está ejecutándose gestionado por el tuner, primero se detendrá sin previo aviso y, a continuación, se eliminará. Tenga en cuenta que este estado de instalación elimina los paquetes que fueron instalados con el estado de persistir. (En una versión anterior, este estado recibía el nombre de eliminar.)
N/D
Principal
Descarga el paquete y lo designa como channel principal del tuner; es decir, como channel que se inicia cada vez que se ejecuta el tuner y a menudo se utiliza como GUI del tuner.
N/D

Prioridad de estados

Pueden producirse conflictos entre los estados de los paquetes cuando un cliente final es miembro de dos o más grupos o cuando se ha asignado una política a un cliente final y a los miembros de un grupo al que pertenece dicho cliente final. En tales casos, se aplican las siguientes reglas de prioridad. Policy Manager aplicará el estado que tenga la prioridad más alta.

  1. Excluir (prioridad más alta)
  2. Desinstalar
  3. Principal
  4. Instalar-iniciar-persistir
  5. Instalar-iniciar
  6. Instalar-persistir
  7. Instalar
  8. Descargar
  9. Anunciar (prioridad más baja)

    10. Nota: Los conflictos se dan entre los estados de los paquetes, no como consecuencia de la manera en que se asignan las políticas. Así pues, la resolución de conflictos no depende de que se asigne un cliente final directa o indirectamente como miembro de un grupo. Para obtener más información, consulte Resolución de conflictos: estados y planificaciones de políticas.
