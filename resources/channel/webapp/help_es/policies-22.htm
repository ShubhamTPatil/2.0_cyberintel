Es posible que existan diferentes paquetes con la misma prioridad de instalación cuando visualice la página Detalles de destino. Supongamos, por ejemplo, que ha establecido las siguientes prioridades de instalación para el usuario Alan:

    • Paquete A, prioridad de instalación 1
    • Paquete B, prioridad de instalación 2
    • Paquete C, prioridad de instalación 3

Asimismo, ha establecido las siguientes prioridades de instalación para el grupo de usuarios Equipo de ingenieros:

    • Paquete D, prioridad de instalación 1
    • Paquete E, prioridad de instalación 2
    • Paquete F, prioridad de instalación 3

Si Alan es miembro del Equipo de ingenieros, la página Detalles de destino mostrará prioridades de instalación que se solapan entre sí (dos 1, dos 2 y dos 3). En este caso, todos los paquetes que tengan la prioridad 1 tendrán prioridad frente a aquellos cuya prioridad de instalación sea 2, etc. Puede resolver otras colisiones de prioridades de instalación utilizando el valor Prioridad inicial para asignar diferentes bloques de números de prioridad a los paquetes con destinos solapados. Véase Configuración de prioridad de instalación de los paquetes.
