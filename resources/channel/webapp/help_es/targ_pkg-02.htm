Puede encontrar los siguientes tipos de destinos:

    • Un usuario (un cliente final identificado por un nombre de inicio de sesión)
    • Una máquina
    • Un grupo de usuarios
    • Un grupo de máquinas
    • Un contenedor o una unidad organizativa (OU)
    • Un dominio en el Active Directory
    • Una colección, que es lo que se obtiene de una base de datos o de una consulta LDAP. Para obtener más información, consulte ¿Qué es una colección?.
    • Todos los clientes finales, que es un grupo virtual especial compuesto por todos los clientes finales. Para obtener más información, consulte ¿Qué es el destino Todos los clientes finales?.

Después de haber asignado un paquete a un destino, puede asignar un estado de instalación a cada paquete, así como las planificaciones y la configuración de propiedades que tengan relación con la instalación de cada cliente final. Toda la información que usted asigna a un destino se llama política.

Resolución jerárquica de políticas. La resolución de políticas de los destinos es jerárquica. Por ejemplo, éste es el cliente final:

cn=usuario1,cn=ordenadores,ou=finanzas,dc=compañía,dc=com

El cliente final recibe políticas asignadas a los siguientes destinos:

    • cn=ordenador,ou=finanzas,dc=compañía,dc=com
    • ou=finanzas,dc=compañía,dc=com
    • dc=compañía,dc=com
