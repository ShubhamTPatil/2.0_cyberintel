Para obtener información sobre la instalación de los componentes mencionados en esta sección, consulte la Guía de implementación e instalación de BMC Configuration Management. Para obtener información sobre la configuración de Deployment Manager y Deployment Service, consulte la Guía del administrador de Server Management; para obtener información sobre la configuración de Report Center y Scanner Service, consulte la Guía del administrador de Report Center. Estos documentos también se encuentran disponibles en el sitio web de soporte al cliente de BMC.

Servidor. Debe tener los siguientes componentes instalados y configurados en la misma máquina en la que haya instalado Policy Manager:

    • Report Center
    • Deployment Manager

      • Nota: BMC recomienda ejecutar Deployment Manager en una máquina diferente de Policy Manager.

Para una nueva instalación de Deployment Manager, debe iniciar sesión y aceptar la licencia antes de utilizarlo con Policy Manager. También necesitará configurar Deployment Manager para la consola (Configuración del sistema > General > Configuración de integración de Deployment Manager) de forma que Policy Manager pueda comunicarse correctamente con Deployment Manager. Necesitará la siguiente información:

    • El nombre de host de la máquina en que se ejecuta Deployment Manager. Esta máquina puede ser la misma en que se está ejecutando Policy Manager , o puede ser otra diferente de la se esté ejecutando Deployment Manager.
    • El número de puerto utilizado por el tuner que ejecuta el channel de Deployment Manager. El número predeterminado es 7717.
    • El nombre y la contraseña de usuario para acceder a Deployment Manager. Deployment Manager debe haber sido configurado para dar acceso a este usuario.
    • El nombre y la contraseña de usuario del administrador remoto para los tuners de clientes finales a los que se conecta Deployment Manager. El nombre y la contraseña de usuario fueron asignados a los tuners de clientes finales en el momento de crear los instaladores del tuner o en el momento de conectarse a ellos a través de Tuner Administrator. Todos los tuners de clientes finales deben utilizar el mismo nombre y contraseña de usuario de administrador remoto.

Existen también configuraciones opcionales que puede establecer en la página Configuración de integración del Deployment Manager. Además, puede iniciar sesión en Deployment Manager para cambiar la configuración predeterminada, incluidos el estado del puerto (el puerto utilizado para contestar los mensajes de estado de los servidores de destino a Deployment Manager). Para obtener más información sobre la configuración de Deployment Manager, consulte la ayuda en línea de Deployment Manager.

Antes de realizar ninguna actualización, necesitará también habilitar la función de actualización inmediata de política, ya que se encuentra deshabilitada de forma predeterminada. Para obtener más información, consulte Habilitar y deshabilitar actualizaciones inmediatas de política.

Cliente. Debe tener los siguientes componentes instalados en los clientes finales junto con Policy Service:

    • Deployment Service
    • Scanner Service

      • Nota: Si desea utilizar la función de actualización inmediata de política con las colecciones, asegúrese de que Scanner Service se ejecute en el cliente final al menos una vez después de haberse creado las colecciones. Ejecutar de nuevo Scanner Service en esos clientes finales asegura que la información de los miembros requerida por la función de actualización inmediata de política esté disponible en la base de datos.
