Le plugin Policy Service est le composant côté serveur de Policy Service. Ce plugin est situé sur le transmitter qui héberge le channel Policy Service. Le plugin se connecte au service d'annuaire afin de définir à quelles cibles une machine cible appartient et effectue une requête auprès du service d'annuaire pour déterminer quelles policies s'appliquent à une machine cible.

Seuls les administrateurs primaires ont l'autorisation d'accéder aux paramètres de configuration de plugin Policy Service. Les administrateurs standard et les opérateurs n'ont pas le droit d'accéder aux paramètres de configuration du plugin, ni de les modifier.

Vous pouvez définir les options du plugin Policy Service pour :

    • Spécifier l'adresse URL du channel Policy Service sur lequel vous souhaitez publier le plugin.
    • Fournir le nom d'utilisateur et le mot de passe de publication exigés pour la publication des modifications de configuration au transmitter qui héberge le plugin (dans le cas où le droit de publication est requis).
    • Spécifier le service d'annuaire que vous souhaitez utiliser.
    • Paramétrer le nombre de connexions entre le plugin et le service d'annuaire.

      • Remarque : pour modifier les paramètres de configuration, vous devez être connecté en tant qu'administrateur primaire. Au cours de l'installation et de la configuration initiales, vous devez spécifier quels utilisateurs sont des administrateurs primaires et lesquels sont des administrateurs standard. Les administrateurs standard ne sont pas habilités à accéder ni à modifier les paramètres de configuration de Policy Manager.

      Pour définir si vous êtes connecté en tant qu'administrateur primaire, placez le curseur de votre souris sur le bouton Statut situé dans le coin supérieur droit de la fenêtre de la console.
