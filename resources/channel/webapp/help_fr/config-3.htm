En général, vous devez configurer et publier le plugin à chaque fois que vous installez ou que vous mettez à niveau le Policy Manager. Cependant, vous devez également republier le plugin dans les situations suivantes :

    • Si vous modifiez les paramètres de la page de configuration du plugin.
    • Si vous modifiez les paramètres avec la commande -setpluginparam.
    • Si vous modifiez les attributs de l'objet du Policy Management à l'aide de la commande -configSet.

      • Remarque : vous n'avez pas besoin de publier le plugin Policy Service à chaque fois que vous modifiez des policies.

Lorsque c'est possible, le Policy Manager extrait des paramètres les informations du service d'annuaire suivantes (Application > Console > Paramètres système > Source de données > Services d'annuaire) et les entre directement dans la page de configuration du plugin :

    • Le hostname et le numéro de port du service d'annuaire
    • Le base DN
    • Le bind DN
    • L'utilisation SSL ou non

Si ces informations figurent déjà sur la page, vous devez confirmer qu'elles constituent les paramètres que vous souhaitez utiliser.

Remarque : si vous avez déjà publié le plugin Policy Service sur un transmitter dont le protocole SSL est activé, redémarrez Policy Manager avant de publier le plugin sur un transmitter dont le protocole SSL n'est pas activé.

Pour configurer et publier le plugin Policy Service

  1. Cliquez sur l'onglet Configuration.
  2. Sur la page Configuration, cliquez sur le lien Plugin.

    3. La page de configuration du plugin s'affiche. La date et l'heure de la dernière publication du plugin figurent sur le côté supérieur gauche de cette page.

  3. Dans la section Transmitter maître, spécifiez l'information relative à la publication du plugin :
    1. Spécifiez l'adresse URL du channel Policy Service sur lequel vous souhaitez publier le plugin.

      2. Remarque : l'adresse URL doit se terminer dans SubscriptionService.

    2. Si un droit de publication est exigé, fournissez le nom d'utilisateur et le mot de passe de publication requis pour la publication des modifications de configuration au transmitter qui héberge le plugin.

      3. Remarque : le nom d'utilisateur et le mot de passe ne sont requis que si vous (ou l'administrateur primaire de votre entreprise) avez configuré les transmitters de sorte qu'ils exigent des noms d'utilisateur et des mots de passe afin que vous puissiez y publier des channels et des packages.

  4. Indiquez les informations nécessaires pour la section Services d'annuaire.

    5. En général, les champs de cette section sont déjà remplis avec des informations du service d'annuaire spécifié pour le CMS. Les seuls champs que vous devez changer concernent le bind DN et le mot de passe du bind DN, si, par exemple, vous souhaitez utiliser un bind DN ayant des droits différents de ceux spécifiés pour le CMS.

    Pour plus d'informations sur les champs de cette page, reportez-vous à la section Page de configuration du plugin : champs du service d'annuaire.

  5. Selon le type de service d'annuaire que vous utilisez, vous pouvez avoir la possibilité de vérifier la connexion entre le Policy Manager et le service d'annuaire en cliquant sur le bouton Valider situé à côté de Tester le(s) serveur(s).

    6. Le Policy Manager établit une connexion au service d'annuaire que vous avez spécifié, en vérifiant que le serveur et le port acceptent les connexions à l'aide du nom d'utilisateur et du mot de passe. Le Policy Manager vérifie ensuite que l'objet Subscription configuration existe. L'objet de configuration Subscription est situé dans le container ConfigObjects, un container enfant du container BMC CM. Cet objet est créé lorsque vous exécutez les scripts d'installation ou de mise à jour LDIF.

    Remarque : il n'est pas nécessaire que le service d'annuaire soit accessible à partir de l'ordinateur sur lequel Policy Manager est installé. Par conséquent, même si ce test échoue, le plugin Policy Service est toujours en mesure d'établir une connexion réussie.

  6. Si vous voulez utiliser Policy Manager avec des machines récemment acquises, cochez la case Permettre l'utilisation de policies comme modèles pour les machines récemment acquises. Pour plus d'informations, reportez-vous au chapitre sur l'intégration de l'installation du tuner à l'approvisionnement du système d'exploitation dans le Guide de déploiement BMC CM, disponible sur le site Web de l’assistance à la clientèle BMC.
  7. Cliquez sur Aperçu

    8. La page Aperçu des modifications du plugin s'affiche. Cette page met l'accent sur les modifications que vous avez effectuées sur la page de configuration du plugin, afin que vous puissiez les vérifier avant de publier le plugin. Par défaut, l'ensemble des paramètres est affiché avec les modifications en surbrillance. Pour que seules les modifications s'affichent, cliquez sur le lien Afficher uniquement les modifications.

  8. Après avoir vérifié vos modifications et, au besoin, les avoir changées, cliquez sur Publier.

    9. Lorsque vous cliquez sur le bouton Publier de cette page, vous publiez le plugin Policy Service du channel Policy Service sur le transmitter puis revenez à la page d'accueil de configuration.

Conseil : si vous souhaitez vérifier que le plugin Policy Service a été publié, utilisez le Transmitter Administrator pour consulter les channels de votre transmitter et développer le channel Policy Service pour consulter ses segments, comme décrit dans la Figure 2-1. Si vous voyez le segment .configurator, cela signifie que vous avez publié le plugin sur le transmitter.
