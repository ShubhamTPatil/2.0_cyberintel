Ce chapitre vous explique comment utiliser Policy Manager avec la solution Patch Management de BMC CM pour déployer des patchs sur les machines cibles.

Les sujets abordés sont les suivants :

  • Pré-requis à l'intégration à Patch Management
  • Définition d'un groupe de patchs
  • Définition d'un état d'attribution de groupe de patch
  • Définition de Patch Service
  • Attribution de groupes de patch aux cibles
  • Simulation de l'installation de patchs
  • Affichage des informations détaillées sur l'installation de groupes de patch
  • Configuration des paramètres de redémarrage pour les installations de patchs
  • Remplacement de la planification des mises à jour de Patch Service pour les machines cibles
  • Exemption de Patch Service de la période d'inactivité
  • Policy compliance pour les groupes de patch

La solution Patch Management de BMC CM vous permet de déterminer les types de patchs les plus appropriés pour les machines de votre entreprise, de regrouper et de configurer les patchs et de les publier sur un transmitter. Ensuite, vous utilisez Policy Manager pour déployer les groupes de patch sur les machines cibles sélectionnées. Pour plus d'informations sur la solution Patch Management de BMC CM, reportez-vous au Guide de l'administrateur de Patch Management, disponible sur le site Web de l'assistance à la clientèle BMC.
