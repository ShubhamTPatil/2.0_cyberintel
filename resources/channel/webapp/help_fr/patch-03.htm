Un groupe de patch est une liste contenant un ou plusieurs ID de patch, chacun servant à identifier de manière univoque un patch, ainsi que d'autres informations complémentaires. Vous pouvez cibler des groupes de patch pour des machines cibles individuelles ou pour des groupes de machines cibles définis par l’utilisateur. Les groupes de patch vous offrent une grande souplesse pour le test et la distribution de patchs. Vous pouvez par exemple créer un groupe de patch de sécurité afin de les distribuer uniquement aux machines cibles utilisées par votre force de vente.

Dans Policy Manager, l'attribution de groupes de patch à des machines cibles est similaire à l'attribution de packages. Vous sélectionnez une ou plusieurs machines cibles, vous modifiez les policies, puis vous ajoutez les groupes de patch que vous souhaitez attribuer.

L'icône Groupe de patch identifie les groupes de patch. Si vous souhaitez savoir quels patchs appartiennent à un groupe de patch, cliquez sur le nom du groupe de patch (qui s'affiche généralement sous forme de lien de couleur bleu).

Remarque : vous pouvez visualiser les patchs membres d'un groupe de patch à condition que ce dernier ait été publié sur le transmitter maître spécifié lors de la configuration de Patch Manager.
