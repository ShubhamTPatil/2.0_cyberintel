Cette rubrique sert de référence et vous aide à terminer la procédure décrite dans la section Configuration des paramètres de redémarrage pour les installations de patchs. La configuration des paramètres de redémarrage concerne les propriétés de tuner suivantes :

    • marimba.reboot.interact — Indique si la machine cible doit afficher un message d'alerte pour avertir les utilisateurs avant le redémarrage. La valeur par défaut est true (un message d'alerte s'affiche).
    • marimba.reboot.interact.allowSnooze — Indique si les utilisateurs sont autorisés à reporter un redémarrage. Si cette propriété a la valeur true, une boîte de dialogue s'affiche, autorisant les utilisateurs à reporter un démarrage. Si elle porte la valeur false, une boîte de dialogue d'alerte apparaît ; elle contient un minuteur affichant le compte à rebours avant le redémarrage. La valeur par défaut est true.
    • marimba.reboot.interact.snooze.maxTime — Indique le délai maximum (en millisecondes) pendant lequel les utilisateurs sont autorisés à reporter un redémarrage. La valeur par défaut est 0 (il n'existe pas de délai maximum).
    • marimba.reboot.interact.timer — Indique la durée, en secondes, de l'attente ou du compte à rebours avant le redémarrage. Cette valeur correspond à l'heure de début affichée dans la boîte de dialogue d'alerte si vous affectez à la propriété marimba.reboot.interact.allowSnooze la valeur false. La valeur par défaut est 60 secondes. Si vous affectez la valeur 0, la machine redémarre immédiatement.
    • marimba.reboot.never — Indique si la machine est redémarrée automatiquement lorsque cela est nécessaire (par exemple après l'installation d'un patch). Affectez la valeur true si vous ne souhaitez pas que la machine redémarre automatiquement.

Vous pouvez afficher ces propriétés dans l'onglet Avancé > Propriétés du tuner/package lors de la modification d'une policy.
