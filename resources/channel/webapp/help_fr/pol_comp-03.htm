La définition de la conformité dépend des éléments suivants :

    • Un package est conforme si l'état sur la machine cible correspond à celui indiqué dans la policy.
    • Une machine cible est conforme si tous les packages qui lui sont directement ou indirectement attribués (dans une policy active) sont conformes.
    • Une cible agrégée (groupe, container, domaine, Toutes les machines cibles) est conforme si tous ses membres sont conformes.
    • Si tous les packages sur la machine cible sont inactifs (la planification d'activation n'a pas encore eu lieu), la machine cible n'est pas incluse dans le calcul des pourcentages, c'est-à-dire qu'elle est ajoutée à l'ensemble des machines dont le statut est Non enregistré.

Pour plus d'informations sur la détermination de la policy compliance, reportez-vous au Guide de l’administrateur de Policy Management, disponible sur le site Web du support BMC Software.
