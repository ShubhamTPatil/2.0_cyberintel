Si vous adoptez la solution BMC Change and Configuration Management (CCM) associée à BMC Business Service Management (BSM), vous pouvez lancer et gérer une tâche de policy à partir du système Action Request (AR) ou du système Configuration Management (CM). Policy Manager vous permet d'exécuter et de suivre les modifications de policy provenant du système AR en associant les tâches de policy à leur ID de modification parent. BMC CM permet également de capturer une vérification à rebours pouvant être consultée dans Report Center.

Le système AR actualise automatiquement le contenu des champs ID de modification et ID de tâche de la page Modification de policy.

Remarque : si vous utilisez une solution Change Management tierce, vous pouvez également saisir manuellement les ID de modification ou les ID de tâche et capturer les données relatives aux modifications et aux tâches dans les fichiers de log.

Le système AR impute, par ailleurs, les informations suivantes lors de l'attribution à BMC Configuration Management d'une tâche de policy associée à une modification :

    • Données cibles disponibles via les relations LDAP vers Configuration Management Database (CMDB)
    • Packages répertoriés comme éléments Software Library (SLI) dans Definitive Software Library (DSL)

Pour plus d'informations sur la solution CCM, reportez-vous à l'annexe Présentation de CCM dans le Guide de l'administrateur de Policy Management, disponible sur le site Web du support BMC Software. Pour plus d'informations sur la configuration de l'intégration à Task Management System, reportez-vous au Guide de l'administrateur de BMC Remedy 7.0 Task Management, disponible sur le site Web de l'assistance à la clientèle BMC.
