Les états d'installation déterminent comment et si un package est installé sur une cible. Pour chaque package attribué à une cible, vous pouvez définir un état primaire. À l'aide de Policy Manager, vous pouvez attribuer des états d'installation à un package dans le cadre de la policy pour une ou plusieurs machines cibles.

Le tableau suivant présente les différents états disponibles pour les packages. Les états primaires mettre en attente et annoncer vous permettent de définir un état secondaire pour contrôler l'installation d'un package. Vous pouvez associer une planification d'activation à l'état secondaire pour indiquer une fenêtre temporelle dans l'avenir pendant laquelle l'installation du package doit avoir lieu dans toute votre organisation, même pour les machines cibles qui ne se connectent pas fréquemment au réseau. L'état secondaire par défaut est installer.

Installer-Persister
Installe un package et n'autorise pas sa suppression même si la machine cible est supprimée d'un groupe cible. Cet état est utile pour l'installation de mises à niveau de logiciel sur le système. Le package peut être quand même supprimé avec l'état désinstaller. (Cet état était appelé abonner-persister dans une version précédente.)

Installer-Démarrer-Persister
Installe un package et n'autorise pas sa suppression même si la machine cible est supprimée d'un groupe cible - comme c'est le cas avec l'état installer-persister. Démarre le logiciel installé après l'installation. Le package démarre chaque fois que Policy Service est exécuté. Si le package est déjà en cours d'exécution, Policy Service ne le redémarre pas.
S/O
Exclure
Exclut un package d'une machine cible même s'il fait partie d'un groupe abonné. Cet état est utile quand vous voulez abonner tous les membres d'un large groupe, à l'exception d'un ou deux, à un package. Cet état a la priorité la plus élevée. (Pour plus d'informations, reportez-vous à la section Priorité des états).
Remarque : pour utiliser l'état exclure, vous devez créer une policy indiquant les machines cibles à exclure (pour qu'elles ne reçoivent pas le package), en plus de la policy qui abonne initialement le groupe le plus large au package.
S/O
Désinstaller
Supprime le package. Si le logiciel installé s'exécute sous le contrôle du tuner, il est d'abord arrêté sans avertissement puis supprimé. Notez que cet état d'installation supprimes les packages installés avec persistance. (Cet état était appelé supprimer dans une version précédente.)
S/O
Primaire
Télécharge le package et le désigne comme le channel primaire du tuner—celui qui est démarré chaque fois que le tuner démarre et sert souvent d'interface graphique pour le tuner.

Priorité des états

Des conflits entre les états de package peuvent arriver quand une machine cible est présente dans au moins deux groupes ou quand une policy a été attribuée à la fois à une machine cible unique et aux membres d'un groupe auquel cette machine cible appartient. Dans de tels cas de figure, les règles de priorité suivantes sont appliquées. Policy Manager applique l'état dont la priorité est la plus élevée.

  1. Exclure (priorité la plus élevée)
  2. Désinstaller
  3. Primaire
  4. Installer-Démarrer-Persister
  5. Installer-Démarrer
  6. Installer-Persister
  7. Installer
  8. Mettre en attente
  9. Annoncer (priorité la plus basse)

    10. Remarque : les conflits se produisent entre les états de package et ne sont pas dus à la manière dont une policy est ciblée. La résolution de conflit ne dépend donc pas du fait qu'une machine cible est ciblée directement ou indirectement en tant que membre d'un groupe. Pour plus d'informations, reportez-vous à la section Résolution de conflit : états et planifications dans les policies.
