Dans la page Modification de policy, vous pouvez définir un état primaire pour chaque package dans la policy. Pour une explication sur les états de package, reportez-vous à la section Présentation des états d'installation.

Les états primaires mettre en attente et annoncer vous permettent de définir un état secondaire pour contrôler l'installation d'un package. Vous pouvez associer une planification d'activation à l'état secondaire pour indiquer une fenêtre temporelle dans l'avenir pendant laquelle l'installation du package doit avoir lieu dans toute votre organisation, même pour les machines cibles qui ne se connectent pas fréquemment au réseau. L'état secondaire par défaut est installer.

Pour plus d'informations sur les états primaires et secondaires, reportez-vous à la section Présentation des états d'installation.

La procédure suivante présuppose que vous vous trouvez dans l'onglet Packages de la page Modification de policy. Pour plus d'informations, reportez-vous à la section Instructions générales pour la création et la modification de policies.

Pour définir les états primaire et secondaire pour un package

  1. Dans l'onglet Packages de la page Modification de policy, trouvez la ligne du package pour lequel vous souhaitez définir des états primaire et secondaire.
  2. Sur la ligne du package, choisissez un état dans la liste déroulante sous la colonne État primaire.
  3. Si un état secondaire est applicable, sélectionnez un état dans la liste déroulante sous la colonne État secondaire. L'état secondaire par défaut est installer.
