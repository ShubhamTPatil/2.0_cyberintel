Vous pouvez définir la planification pour les états primaire et secondaire des packages. Quand une installation est planifiée, le tuner cible télécharge le package du transmitter et accomplit l'activité d'installation que vous avez indiquée. Si vous indiquez un état primaire et un état secondaire, le tuner cible accomplit les activité d'installation à l'heure planifiée. Les planifications sont déterminées en fonction du fuseau horaire de la machine cible.

Cette procédure présuppose que vous vous trouvez dans l'onglet Packages de la page Modification de policy. Pour plus d'informations, reportez-vous à la section Instructions générales pour la création et la modification de policies.

Pour définir la planification pour l'état primaire et l'état secondaire d'un package

  1. Dans l'onglet Packages de la page Modification de policy, trouvez la ligne du package pour lequel vous voulez définir des planifications.
  2. Sur la ligne du package, cliquez sur la flèche dans la première colonne dans la partie gauche de la page afin d'afficher les informations de planification pour le package.
  3. Pour définir une planification pour l'état primaire :
    1. Dans la section Planification pour l'état primaire, cliquez sur Modifier.

      2. La page Modification de planification primaire apparaît.

    2. Dans les zones Activation et Expiration, indiquez les dates et heures définissant la fenêtre temporelle pendant laquelle vous désirez rendre des packages disponibles pour l'état primaire.

      3. Pour plus d'informations, reportez-vous à la section Définition de l'activation et de l'expiration.

    3. Cliquez sur OK pour enregistrer la planification et revenir à la page Modification de policy.

      4. Remarque : la planification apparaît désormais dans la section Planification pour l'état primaire.

  4. Si vous avez indiqué un état secondaire pour le package, définissez une planification pour l'état secondaire :
    1. Dans la section Planification pour l'état secondaire, cliquez sur Modifier.

      2. La page Modification de planification secondaire apparaît.

    2. Dans les zones Activation et Expiration, indiquez les dates et heures définissant la fenêtre temporelle pendant laquelle vous désirez rendre des packages disponibles pour l'état secondaire.

      3. Pour plus d'informations, reportez-vous à la section Définition de l'activation et de l'expiration.

    3. Cliquez sur OK pour enregistrer la planification et revenir à la page Modification de policy.

      4. Notez que la planification apparaît désormais dans la section Planification pour l'état secondaire.

Lorsque vous avez terminé de modifier la policy, affichez l'aperçu, puis enregistrez-la. Pour plus d'informations, reportez-vous à la section Aperçu et enregistrement des modifications de policy.

Remarque : vous ne pouvez pas définir une planification primaire ni une planification secondaire si cette dernière se produit plus tôt que la planification primaire. Par exemple, les planifications suivantes ne sont pas valides :

Planification primaire : activation le 2/1/04 à 1h30

Planification secondaire : activation le 1/1/04 à 12h00
