Vous pouvez définir des planifications communes à plusieurs packages pour que deux ou plusieurs packages dans une policy aient les mêmes planifications. Quand vous définissez des planifications communes pour plusieurs packages, vous définissez toutes les planifications : primaire, secondaire, mise à jour et réparation. Même si vous modifiez un seul type de planification, les autres planifications affichées dans la page Définition de planification commune sont appliquées à tous les packages.

Cette procédure présuppose que vous vous trouvez dans l'onglet Packages de la page Modification de policy. Pour plus d'informations, reportez-vous à la section Instructions générales pour la création et la modification de policies.

Pour définir des planifications communes à plusieurs packages

  1. Dans l'onglet Packages de la page Modification de policy, cochez les cases correspondant aux packages pour lesquels vous souhaitez définir des planifications communes.
  2. Cliquez sur Définir une planification commune.
  3. Dans la page Définition de planification commune, définissez les planifications primaire, secondaire, de mise à jour et de réparation pour les packages.

    4. Par exemple, pour définir la planification pour l'état primaire :

    1. Dans la section Planification pour l'état primaire, cliquez sur Modifier.

      2. La page Modification de planification primaire apparaît.

    2. Dans les zones Activation et Expiration, indiquez les dates et heures définissant la fenêtre temporelle pendant laquelle vous désirez rendre des packages disponibles pour l'état primaire.
    3. Cliquez sur OK pour enregistrer la planification et revenir à la page Modification de policy.

      4. Notez que la planification apparaît désormais dans la section Planification pour l'état primaire.

      Pour plus d'informations, reportez-vous aux sections suivantes :

    4. Définition de la planification primaire et secondaire pour les packages
    5. Définition de la planification de mise à jour pour les packages
    6. Définition de la planification de réparation pour les packages
  4. Une fois la définition des planifications terminée, vérifiez les planifications affichées dans la page Définition de planification commune.

    5. Remarque : les planifications affichées dans la page Définition d'une planification commune sont appliquées à tous les packages sélectionnés. Ceci est valable même si vous n'avez pas modifié un type de planification (dans l'hypothèse où vous avez par exemple modifié la planification primaire mais pas la planification de mise à jour).

  5. Après avoir vérifié les planifications, cliquez sur OK pour enregistrer la planification et revenir à la page Modification de policy.

Lorsque vous avez terminé de modifier la policy, affichez l'aperçu, puis enregistrez-la. Pour plus d'informations, reportez-vous à la section Aperçu et enregistrement des modifications de policy.
