Lorsque les états ou les planifications spécifiés pour les packages sont différents pour les cibles dans une policy, vous devez indiquer si vous voulez maintenir cette différence ou utiliser un paramètre pour toutes les cibles.

Si des différences appellent à une résolution, le message suivant apparaît dans la page Modification de policy :

Les packages et les planifications signalés par ont des états et des planifications différents sur les cibles sélectionnées. Vous pouvez conserver ces différences ou sélectionner un état ou une planification à appliquer à toutes les cibles sélectionnées.

Pour résoudre les différences d'états et de planifications d'un package

  1. Dans la page Modification de policy, trouvez les packages qui ont des états ou des planifications différents pour les cibles sélectionnées.
  2. Dans l'onglet Packages, sous la colonne État primaire ou État secondaire, sélectionnez l'une de ces options :
    • Conserver la différence pour les cibles—Sélectionnez cette option si vous voulez conserver les différents états de package pour les cibles, comme par exemple l'état mettre en attente pour une cible et l'état installer pour une autre.
    • <État>—Sélectionnez cette option si vous voulez spécifier un état de package à appliquer à toutes les cibles.
  3. Dans la section Planification, pour les planifications qui sont différentes pour les cibles sélectionnées, sélectionnez l'une de ces options :
    • Conserver la différence de planification <primaire, secondaire, mise à jour, réparation> pour les cibles—Sélectionnez cette option si vous voulez conserver des planifications de package différentes pour les cibles, comme si vous vouliez par exemple qu'un package soit disponible pour l'installation à différentes dates pour chaque cible.
    • Appliquer la planification <primaire, secondaire, mise à jour, réparation> modifiée à toutes les cibles sélectionnées—Sélectionnez cette option si vous voulez indiquer une planification pour le package à appliquer à toutes les cibles. Vous devez indiquer la planification que vous voulez appliquer. Pour plus d'informations, reportez-vous à la section Présentation des planifications de package.

Lorsque vous avez terminé de modifier la policy, affichez l'aperçu, puis enregistrez-la. Pour plus d'informations, reportez-vous à la section Aperçu et enregistrement des modifications de policy.
