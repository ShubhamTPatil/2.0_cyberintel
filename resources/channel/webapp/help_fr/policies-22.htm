Il est possible d'avoir la même priorité d'installation pour différents packages quand vous affichez la page Détails de la cible. Par exemple, vous définissez les priorités d'installation suivantes pour l'utilisateur Alain :

    • Package A, priorité d'installation 1
    • Package B, priorité d'installation 2
    • Package C, priorité d'installation 3

Vous définissez aussi les priorités d'installation suivantes pour le groupe d'utilisateurs Service technique :

    • Package D, priorité d'installation 1
    • Package E, priorité d'installation 2
    • Package F, priorité d'installation 3

Si Alain est un membre du service technique, la page Détails de la cible affiche des priorités d'installation qui se chevauchent (deux 1, deux 2 et deux 3). Dans ce cas, tous les packages avec la priorité d'installation 1 ont la priorité sur ceux avec la priorité d'installation 2, etc. Les conflits de priorité d'installation peuvent également être résolus en utilisant la valeur Priorité de démarrage pour l'attribution de blocs différents de numéros de priorité à des packages dont les cibles se chevauchent. Reportez-vous à la section Définition de la priorité d'installation des packages.
