Si vous ne respectez pas les recommandations, des conflits de propriétés de tuner et de channel peuvent survenir lorsque des valeurs différentes sont définies pour une même propriété dans des groupes différents. Cette section vous explique comment résoudre les conflits de valeurs de propriété. Pour plus d'informations sur le paramétrage des propriétés de tuner et de package, reportez-vous à la section Paramétrage des propriétés de tuner et de package d'une cible.

Propriétés de tuner. Si vous avez inclus une machine cible en tant que membre de plusieurs groupes, vous pouvez paramétrer différentes valeurs pour une seule propriété de tuner sur chaque policy du groupe. Étant donné que les conflits de propriétés de tuner ne peuvent pas être résolus à l'aide d'un schéma de priorité, il est impossible de prédire quelle valeur prévaut.

BMC recommande de définir les propriétés de tuner de manière uniforme pour tous les groupes auxquels appartient une machine cible.

Propriétés du package. Si une propriété de package est à la fois définie pour tous les packages abonnés et un package spécifique, la valeur de propriété du package spécifique est prioritaire. Par exemple, supposons les paramètres de propriété suivants :

semisilent=false pour tous les packages abonnés
(La syntaxe est semisilent,subscribers=false.)

semisilent=true pour le package WordProcessorApp
(La syntaxe est semisilent,http://trans.entreprise.com:5282/Applications/WordProcessorApp=true.)

Dans ce cas, la valeur de la propriété semisilent du package WordProcessorApp est définie sur true. Si la propriété semisilent est Null pour d'autres packages abonnés, Policy Manager la définit sur false.
