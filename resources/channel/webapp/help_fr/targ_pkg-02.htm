Une cible peut être :

    • Un utilisateur (une machine cible identifiée par un nom de connexion)
    • Une machine
    • Un groupe d'utilisateurs
    • Un groupe de machines
    • Un container ou une unité organisationnelle (OU)
    • Un domaine d'Active Directory
    • Une collection, résultant d'une requête de la base de données ou d'une requête LDAP. Pour plus d'informations, reportez-vous à la section Définition d'une collection.
    • Toutes les machines cibles, un groupe virtuel spécifique composé de toutes les machines cibles. Pour plus d'informations, reportez-vous à la section Définition de la cible Toutes les machines cibles.

Une fois que vous attribuez un package à une cible, vous pouvez affecter un état d'installation à chaque package, ainsi que des planifications et des paramètres de propriété affectant l'installation de chaque machine cible. L'ensemble des informations que vous attribuez à une cible est nommé sa policy.

Résolution hiérarchique de policies. La résolution de policies pour des cibles est hiérarchique. Prenons par exemple la machine cible suivante :

cn=utilisateur1,cn=ordinateurs,ou=finance,dc=entreprise,dc=com

Cette machine cible fait attribuer les policies aux cibles suivantes :

    • cn=ordinateur,ou=finance,dc=entreprise,dc=com
    • ou=finance,dc=entreprise,dc=com
    • dc=entreprise,dc=com
