La partie gauche de la page Vue cible affiche les cibles auxquelles vous voulez attribuer des packages. Lorsque vous sélectionnez une cible, vous pouvez afficher les packages leur ayant été affectés directement et indirectement (côté droit de la page).

Vous pouvez consulter les cibles disponibles en navigant sur la partie gauche de la page. La partie gauche de la page répertorie les cibles : les utilisateurs et machines spécifiques, les groupes d'utilisateurs et de machines et les collections résultant de requêtes de base de données ou LDAP.

Lorsque vous consultez cette page pour la première fois, la page d'accueil de la cible, composée de containers de niveau supérieur du service d'annuaire dans lesquels les informations sont stockées, s'affiche. Par exemple, dans Active Directory, vous pouvez généralement voir les domaines racines de l'environnement forêt. Vous pouvez développer les domaines racines afin d'afficher les objets qui peuvent être utilisés en tant que cibles dans ce domaine. Pour plus d'informations, reportez-vous au chapitre sur Active Directory dans le Guide de l’administrateur de Policy Management, disponible sur le site Web du support BMC Software.

Les icônes de cible de la page Vue cible indiquent comment Policy Manager obtient sa liste de cibles :

    • Domaines : icône de domaine qui peut être développée et qui indique que les cibles sont en cours d'obtention à partir d'un environnement forêt d'Active Directory. Cette icône représente les domaines racines et les domaines subordonnés.
    • Containers : icône de dossier qui peut être développée et qui indique que les cibles sont en cours d'obtention à partir du service d'annuaire. Cette icône représente les unités organisationnelles et les containers.
    • Utilisateurs (provenant du transmitter) : icône du transmitter qui peut être développée et qui indique que les utilisateurs et les groupes d'utilisateurs sont obtenus à partir du transmitter sur lequel vous avez publié le plugin Policy Service (à l'aide de la page Configuration du plugin).

      • Remarque : si vous (ou un administrateur primaire) n'avez pas encore publié le plugin Policy Service sur le transmitter, Policy Manager ignore quel transmitter utiliser ; plugin Policy Service ne peut alors pas obtenir les informations sur les utilisateurs et groupes d'utilisateurs. De plus, si le transmitter dispose d'un accès restreint, vous devez paramétrer le nom d'utilisateur et le mot de passe à utiliser pour une authentification à l'aide du transmitter en exécutant l'option de ligne de commande suivante :

      -txadminacess <nom_utilisateur> <mot_de_passe>

      Le nom d’utilisateur et le mot de passe que vous spécifiez avec cette option sont ceux qui sont nécessaires au transmitter. Vous devez quand même spécifier le nom d'utilisateur et le mot de passe requis pour l'authentification par Policy Manager.

      Pour plus d'informations sur les options de ligne de commande, reportez-vous à la section sur la ligne de commande dans le Guide de l’administrateur de Policy Management, disponible sur le site Web du support BMC Software.

De plus, la cible Toutes les machines cibles s'affiche. Il s'agit d'une cible virtuelle contenant toutes les machines cibles de votre site sur lesquelles Policy Service est installé.

Si vous explorez un environnement forêt à plusieurs domaines (Active Directory), vous pouvez explorer les arbres et les domaines.
