Deux options sont disponibles lors de la recherche de cibles :

    • Recherche de base : cette option permet la recherche de cibles à l'aide du nom commun (CN). Vous pouvez également définir un type pour limiter votre recherche. Pour plus d'informations, reportez-vous à la section Recherche de base de cibles.
    • Recherche avancée : les utilisateurs avancés familiarisés avec les requêtes et les filtres de recherche du LDAP peuvent rechercher des cibles de façon plus flexible grâce à cette option. Pour plus d'informations, reportez-vous à la section Recherche avancée de cibles.

Vous pouvez rechercher des cibles dans le groupe de cibles ou le container actuellement développé. Cette fonction est utile lorsque vous employez des containers composés d'un grand nombre de sous-containers ou d'unités organisationnelles contenant eux-mêmes un grand nombre de membres.

Remarque : Active Directory et ADAM n'obtiennent pas le nombre total d'entrées tant que vous n'avez pas fait défiler l'ensemble des résultats de la recherche. Par conséquent, la liste déroulante affiche « ... » pour le nombre total d'entrées, jusqu'à ce que vous ayez atteint la dernière page.

Recherche de base de cibles

Pour effectuer une recherche de base de cibles

  1. Dans la partie gauche de la page Vue cible, cliquez sur le container dans lequel vous souhaitez effectuer une recherche.
  2. Dans la partie gauche de la page Vue cible, cliquez sur le lien Recherche de base.

    3. Zone Recherche de base s'affiche.

  3. Vous devez fournir les informations suivantes :
    • Dans la zone de texte Rechercher, entrez le nom commun (CN) ou une partie du CN d'une cible.

      • Vous n'avez pas besoin de connaître le nom complet de la cible. Vous pouvez utiliser l'astérisque (*) en tant que caractère de remplacement pour zéro ou plusieurs caractères du nom. Si un astérisque est présent dans le nom, vous devez le faire précéder d'un caractère d'échappement pour qu'il ne soit pas considéré comme un caractère générique. Pour plus d'informations, reportez-vous à la section Caractères spéciaux dans les chaînes de recherche.

      Remarque : l'opération de recherche ne fait aucune distinction entre les majuscules et les minuscules.

    • Dans la liste Limiter à, sélectionnez le type de cible sur lequel vous faites une recherche, ou choisissez Tous types.
    • Pour mieux restreindre votre recherche, vous pouvez cocher la case Ne pas rechercher dans les sous-containers.
  4. Cliquez sur Valider.

Les résultats de la recherche sont affichés sous la zone de recherche. Lorsque les résultats sont affichés dans la liste, l'étiquette du fil d'Ariane se modifie afin d'indiquer que vous consultez les résultats de la recherche.

Pour revenir à la vue Cible de niveau supérieur, cliquez sur l'onglet Page d'accueil.

Pour revenir à la vue sur laquelle vous avez initialement démarré votre recherche, cliquez sur l'onglet Vue cible.

Recherche avancée de cibles

Utilisez la recherche avancée si vous êtes familiarisé avec les requêtes et les filtres de recherche du LDAP du service d'annuaire que vous utilisez. Du fait que Policy Manager transmet la requête directement au service d'annuaire, cette recherche est plus rapide que la recherche de base. Vous pouvez entrer des requêtes simples, comme par exemple la recherche de cibles avec pour CN utilisateurs.

cn=utilisateurs

Vous pouvez entrer plusieurs requêtes complexes qui utilisent ces conditions, telles que la recherche de cibles avec un CN comportant utilisateurs ou groupes à l'aide de l'annotation ou (|) :

(|(cn=utilisateurs) (cn=groupes))

Pour obtenir plus d'informations concernant les filtres de recherche du LDAP, reportez-vous à l'adresse http://www.ietf.org/rfc/rfc2254.txt.

Remarque : vous ne pouvez pas employer la recherche avancée pour rechercher des cibles à l'aide du nom distinctif complet (DN).

Pour effectuer une recherche avancée de cibles

  1. Dans la partie gauche de la page Vue cible, cliquez sur le container dans lequel vous souhaitez effectuer une recherche.
  2. Dans la partie gauche de la page Vue cible, cliquez sur le lien Recherche avancée.

    3. La zone Recherche avancée s'affiche.

  3. Dans la zone de texte Requête LDAP, entrez un filtre de recherche du LDAP.

    4. Vous pouvez entrer des requêtes simples ou complexes que Policy Manager transmets directement au service d'annuaire. Si votre recherche comporte un caractère spécial (par exemple *,(,),\), reportez-vous à la section Caractères spéciaux dans les chaînes de recherche.

    Remarque : L'opération de recherche ne fait aucune distinction entre les majuscules et les minuscules.

  4. Pour mieux restreindre votre recherche, vous pouvez cocher la case Ne pas rechercher dans les sous-containers.
  5. Cliquez sur Valider.

Les résultats de la recherche sont affichés sous la zone de recherche. Lorsque les résultats sont affichés dans la liste, l'étiquette du fil d'Ariane se modifie afin d'indiquer que vous consultez les résultats de la recherche.

Pour revenir à la vue Cible de niveau supérieur, cliquez sur l'onglet Page d'accueil.

Pour revenir à la vue sur laquelle vous avez initialement démarré votre recherche, cliquez sur l'onglet Vue cible.
