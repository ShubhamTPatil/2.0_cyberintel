La page Détails de la cible présente tous les packages attribués à la cible ou aux cibles sélectionnée(s), ainsi que la priorité d'installation, les états primaire et secondaire et les planifications de chaque cible.

Pour afficher les détails d'une cible (ou de plusieurs cibles si vous êtes en mode de sélection multiple)

  1. Sur la page Vue cible, sélectionnez une ou plusieurs cibles si vous êtes en mode de sélection multiple.
  2. Cliquez sur le lien Vue détaillée pour afficher les détails des packages attribués aux cibles.

    3. La page Détails de la cible s'affiche. Cette page présente les packages attribués à la cible ou aux cibles sélectionnée(s), ainsi que la priorité d'installation, les états primaire et secondaire et les planifications de chaque cible.

    Vous pouvez visualiser les détails des packages sélectionnés ou modifiez leur attribution aux cibles. Cliquez sur le nom de la colonne pour effectuer le tri par contenu de colonne. Faites défiler les longues listes en cliquant sur les liens précédent et suivant.

  3. Pour revenir à la page Vue cible, cliquez sur le lien Vue basique.

Tri de la liste des packages

Sur la page Détails de la cible, vous pouvez trier la liste des packages selon :

    • Les planifications
    • Nom
    • La priorité d'installation (disponible uniquement en mode de sélection unitaire)
    • La cible à laquelle le package est directement attribué (disponible uniquement en mode de sélection unitaire).
    • Le type (comme indiqué par l'icône, par exemple pour un package ou pour un groupe de patch)

Pour effectuer le tri de la liste des packages

Sur la page Détails de cible, cliquez sur le nom de la colonne pour trier les packages à l'aide de cette colonne (par ordre croisant ou décroissant).

    • Si vous effectuez votre tri en fonction des colonnes de planification, les packages sont organisés selon la date et l'heure d'activation de cette planification. Par exemple, si vous effectuez votre tri en fonction de la planification primaire, les packages sont organisés selon la date et l'heure paramétrées pour la planification primaire.
    • Si vous effectuez le tri en fonction du nom du package, les packages sont organisés par ordre alphabétique.
    • Si vous effectuez votre tri en fonction de la priorité d'installation, les packages sont organisés selon la priorité avec laquelle ils doivent être installés. Si vous souhaitez modifier la priorité avec laquelle les applications sont installées, reportez-vous à la section Définition de la priorité d'installation pour les packages dans une policy.
    • Si vous effectuez votre tri en fonction de la cible à laquelle le package est directement attribué, les packages attribués directement sont situés en haut de la liste, suivis des packages attribués indirectement.
