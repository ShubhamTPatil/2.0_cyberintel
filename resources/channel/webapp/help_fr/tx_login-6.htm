Les profils contiennent des paramètres de configuration et non des binaires de produits, pour les composants de l'infrastructure BMC CM. Vous pouvez créer plusieurs profils, un pour chaque type de composant, notamment les proxies, les transmitters, les nœuds gérés (machines cibles), les miroirs et les répéteurs. Chaque profil est enregistré sous la forme d'un segment d'Infrastructure Service. Si vous modifiez un paramètre de profil, la prochaine fois qu'une machine cible met à jour son channel Infrastructure Service, elle obtient les nouveaux paramètres.

Lorsque vous sélectionnez un profil et que vous l'attribuez à une machine cible à l'aide de Policy Manager, la propriété de tuner marimba.tuner.update.profile de la machine cible est définie sur le nom du profil que vous avez spécifié.

Pour plus d'informations sur la création de profils et de types de profils, reportez-vous à l'aide de Configuration et déploiement.
