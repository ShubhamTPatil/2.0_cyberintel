Si vous souhaitez attribuer un profil différent à une cible (ou attribuer un profil à une cible qui n'en possède pas), vous pouvez effectuer cette modification dans la policy de la cible. Pour plus d'informations sur la création et la modification de profils, reportez-vous à l'aide de Configuration et déploiement.

Si vous modifiez le profil sur des cibles, prenez en considération les éléments suivants :

    • Vous attribuez un nouveau profil aux cibles à l'aide de l'onglet Profils de Policy Manager, ou en paramétrant la propriété de tuner marimba.tuner.update.profile, via Tuner Administrator. Ce document explique comment utiliser Policy Manager pour attribuer un nouveau profil. Lorsque vous utilisez Policy Manager pour attribuer un profil, vous pouvez exploiter les collections. Une collection est une liste des utilisateurs ou machines qui répondent à un certain nombre de critères que vous spécifiez dans une requête LDAP ou de base de données. Une fois la requête exécutée, vous pouvez utiliser Policy Manager pour attribuer un profil à la liste (collection) générée par la requête.
    • Le channel Infrastructure Service s'exécute conformément à une planification. Lors de sa prochaine exécution sur les machines, Infrastructure Service télécharge toutes les modifications disponibles et les applique aux paramètres du profil et aux binaires du tuner. Certaines modifications sont appliquées aux machines après le redémarrage des tuners qui y sont hébergés. Pour plus d'informations sur la mise à jour des binaires du tuner, reportez-vous au Guide de mise à niveau de BMC Configuration Management, disponible sur le site Web de l'assistance à la clientèle BMC.

Avant d'attribuer la policy, veillez à exécuter les tâches suivantes :

    • Si le profil que vous souhaitez attribuer à une cible n'existe pas, créez-le.
    • Si vous souhaitez attribuer le profil à une collection, définissez la collection comprenant l'ensemble des machines auxquelles vous souhaitez attribuer le nouveau profil.
    • Créez une requête et spécifiez les critères pour l'ensemble de machines de votre choix. Pour plus d'informations, reportez-vous à la section Définition d'une collection.
    • Exécutez la requête, soit en configurant une planification, soit manuellement.

      • Une fois la requête exécutée, la liste (collection) qui en résulte apparaît en tant que cible dans Policy Manager.

      Remarque : vous pouvez définir des paramètres au niveau de la cible, comme le profil, pour une seule cible à la fois. En revanche, si vous êtes en mode de sélection multiple et que vous modifiez des policies pour plusieurs cibles à la fois, la définition de paramètres au niveau de la cible n'est pas autorisée. Vous pouvez toutefois définir des paramètres au niveau de la cible pour l'ensemble des cibles en sélectionnant la cible Toutes les machines cibles.

Cette procédure présuppose que vous vous trouvez dans l'onglet Packages de la page Modification de policy. Pour plus d'informations, reportez-vous à la section Instructions générales pour la création et la modification de policies.

Pour modifier le profil pour une cible

  1. Dans la page Modification de policy, pour la cible sélectionnée, cliquez sur l'onglet Avancé > Profil.

    2. Si un profil a déjà été attribué à la cible, le nom du profil et l'URL pour le channel Infrastructure Service (la source du profil) apparaissent dans cet onglet. Si aucun profil n'a été attribué à la cible, aucun nom de profil n'apparaît.

  2. Dans l'onglet Profil, cliquez sur Modifier.

    3. La page Choix d'un profil s'affiche. Si la cible ne possède pas de profil, Policy Manager affiche les profils dans le premier channel Infrastructure Service qu'il trouve sur le transmitter à partir duquel le channel Policy Manager est abonné. Par exemple, si le channel Policy Manager est abonné à partir de http://trans.company.com:5282/Marimba/Current/SubscriptionManager, il affiche les profils situés dans le premier channel Infrastructure Service qu'il trouve sur le même transmitter, par exemple http://trans.company.com:5282/Marimba/Current/InfrastructureService.

  3. Si vous souhaitez afficher des profils situés ailleurs, entrez l'URL du transmitter (par exemple http://trans.entreprise.com:5282) puis, si des droits d'accès ont été définis pour le transmitter, entrez le nom d'utilisateur et le mot de passe pour l'abonnement aux channels. Cliquez sur Valider.

    4. Une liste de profils s'affiche dans la partie gauche de la page. Ces profils sont disponibles avec l'Infrastructure Service situé sur le transmitter que vous avez spécifié. Chaque profil est enregistré sous la forme d'un segment dans le channel Infrastructure Service. Par conséquent, les profils figurant dans la liste dépendent de l'Infrastructure Service que vous affichez.

  4. Cliquez sur un profil pour l'afficher dans la partie droite de la page, sous Profil sélectionné, puis cliquez sur OK.

    5. Vous revenez à l'onglet Profil, qui affiche le profil sélectionné.

  5. Lorsque vous avez terminé de modifier la policy, affichez l'aperçu, puis enregistrez-la. Pour plus d'informations, reportez-vous à la section Aperçu et enregistrement des modifications de policy.

L'exécution de cette procédure permet de définir les propriétés de tuner suivantes pour les machines cibles :

marimba.tuner.update.profile

La valeur de cette propriété est le nom du nouveau segment de profil que vous souhaitez attribuer aux machines cibles. Le nom d'un segment de profil a le format suivant : .profil_<nom>

Exemple : marimba.tuner.update.profile=.profil_ny_miroirs

Cette propriété de tuner est définie dans le fichier prefs.txt lors de la prochaine exécution de Policy Service sur une machine. Le nouveau profil est appliqué à la machine lors de la prochaine exécution d'Infrastructure Service sur cette machine, et après le redémarrage du tuner.
