Côté serveur. Les composants suivants doivent être installés et configurés sur la même machine que Policy Manager :

    • Report Center
    • Deployment Manager

      • Remarque : BMC recommande d'exécuter Deployment Manager sur une machine différente de celle qui exécute Policy Manager.

Pour une nouvelle installation de Deployment Manager, vous devez d'abord vous connecter et accepter la licence si vous souhaitez l'utiliser avec Policy Manager. Vous devez également configurer les paramètres de Deployment Manager relatifs à la console (Paramètres système > Général > Paramètres d'intégration de Deployment Manager) afin que Policy Manager puisse communiquer correctement avec Deployment Manager. Vous avez besoin des informations suivantes :

    • Le hostname de la machine sur laquelle s'exécute Deployment Manager. Il peut s'agir de la même machine sur laquelle s'exécute Policy Manager, ou d'une autre machine.
    • Le numéro de port utilisé par le tuner qui exécute le channel Deployment Manager. La valeur par défaut est 7717.
    • Le nom d'utilisateur et le mot de passe permettant d'accéder à Deployment Manager. Deployment Manager doit être configuré pour autoriser l'accès de cet utilisateur.
    • Le nom d'utilisateur et le mot de passe pour l'administration à distance des tuners de machine cible auxquels Deployment Manager se connecte. Le nom d'utilisateur et le mot de passe sont attribués aux tuners de machine cible, soit lors de la création des programmes d'installation des tuners, soit lors de la connexion aux tuners avec Tuner Administrator. Tous les tuners de machine cible doivent utiliser les mêmes nom d'utilisateur et mot de passe pour l'administration à distance.

Il existe également des paramètres facultatifs que vous pouvez configurer dans la page Paramètres d'intégration de Deployment Manager. Par ailleurs, vous pouvez vous connecter à Deployment Manager pour modifier ses paramètres par défaut, y compris le port de statut (port utilisé par les serveurs cibles pour renvoyer des messages de statut à Deployment Manager). Pour plus d'informations sur les paramètres de Deployment Manager, reportez-vous à l'aide en ligne de Deployment Manager.

Pour pouvoir effectuer des mises à jour, vous devez d'abord activer la fonction de mise à jour immédiate de la policy, qui est désactivée par défaut. Pour plus d'informations, reportez-vous à la section Activation et désactivation des mises à jour immédiates de la policy.

Côté client. Les composants suivants doivent être installés sur les machines cibles avec Policy Service :

    • Deployment Service
    • Scanner Service

      • Remarque : si vous souhaitez utiliser la fonction de mise à jour immédiate de la policy avec des collections, assurez-vous que Scanner Service s'exécute au moins une fois sur les machines cibles après la création des collections. Le fait d'exécuter une nouvelle fois Scanner Service sur les machines cibles permet de garantir que les informations d'appartenance requises par la fonction de mise à jour immédiate de la policy sont disponibles dans la base de données.
